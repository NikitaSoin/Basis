## Продукт
Basis — независимый аналитический слой и «второе мнение» для частного инвестора
(не брокер, не сделки, не «купить/продать»; снижает неопределённость, не хайп).
3 модуля: Рынок (карточки компаний + классы активов), Обозреватель, Портфель.
Стек: React (CRA, чистый JS, `/frontend/Basis`, почти весь UI в `src/App.js`),
FastAPI+PostgreSQL+SQLAlchemy+Alembic (`/backend`), деплой git push → Timeweb.

## 🔴 ГЛАВНОЕ ПРАВИЛО — ГОТОВО = РАБОТАЕТ НА ЖИВОМ САЙТЕ
«Код запушен» ≠ готово. Готово = ты открыл inbasis.ru и УВИДЕЛ работающий
результат: данные в БД → бэк отдаёт (проверь запросом к БОЕВОМУ адресу
nikitasoin-basis-e726.twc1.net, не localhost) → фронт рендерит → задеплоено.
- Деплой не мгновенный: после push ждать сборку, проверять смену бандла
  `main.<hash>.js` / опрос боевого эндпоинта (до 10 мин). Не суди «сломано»
  сразу после push.
- Если задача упирается в то, что данные не доходят до боя — это НЕ «готово».
  Доведи путь данных до автоматического деплоя (см. задачу про import) или, если
  не можешь, ЧЕСТНО пометь «на бою НЕТ, нужна команда X», а не ставь «✅ деплой».
- НИКОГДА не отчитывайся «сделано/деплой», если на живом сайте этого нет.

## 🔴 ОТК — СМОТРИ НА ЭКРАН ГЛАЗАМИ, А НЕ НА ЗАМЫСЕЛ
Перед тем как считать задачу готовой, прогони ОТК по тому, КАК ЭТО ВЫГЛЯДИТ НА
ЖИВОМ САЙТЕ (не по коду, не по тексту-замыслу):
- `user-persona-reviewer` — понятно ли, ЧИТАЕТСЯ ли (шрифты, длина, каша),
  отвечает ли на «что мне с этим делать». Смотрит ВЕСЬ экран/вкладку целиком и
  сверяет с соседними вкладками (иконки, переходы, заполненность, единые термины).
- `basis-design-reviewer` — дизайн-конституция (для UI).
- Мнение персоны оценивай критично (может ошибиться из-за неполного обзора —
  не ломай правильное), но РЕАЛЬНЫЕ дефекты (нечитаемо, прочерки, бардак) —
  чини САМ до возврата. Нечитаемая простыня текста = дефект, заворачивай сам.
Если результат нельзя посмотреть на бою (данные не задеплоены) — ОТК неполон,
сначала доведи до боя.

## 🔴 НЕ ПРОСТАИВАЙ — РАБОТАЙ ПО ОЧЕРЕДИ
Владелец отсутствует часами. Сделал одну задачу — НЕ вставай ждать его. Бери
следующую из очереди (ниже), что НЕ требует владельца, и работай дальше.
Останавливайся совсем ТОЛЬКО когда исчерпал всё, что можно без него.
- Веди несколько фронтов: по одному ждёшь ОК — двигай другое.
- Лучше сделать НОВОЕ вчерне (потом владелец поправит), чем простаивать. Но
  «вчерне» ≠ «без ОТК» и ≠ «нечитаемо» — базовое качество и читаемость
  обязательны всегда.
- К владельцу выноси ТОЛЬКО: продуктовые развилки (осознанный выбор стратегии),
  то что требует его доступа/оплаты, и изменения в заповеднике. Технику (деплой,
  формат данных, выбор источника) решай САМ.
- Не накапливай гору развилок как повод встать — если можешь двигать очередь,
  двигай, развилки копи в журнал и продолжай.

### Очередь задач (бери сверху вниз; детали в docs/status.md)
0. ПЕРВОЕ: авто-деплой данных на бой (чтобы не зависеть от ручной команды) — без
   этого ты не можешь доводить задачи до «готово». См. задачу владельца.
1. Облигации вглубь: РЕАЛЬНЫЙ анализ ВСЕХ (не 3 примера) — связка эмитент,
   долговая нагрузка «расплатится ли», доходность vs риск (ВДО), риски
   управления/гео эмитента; сортировка по реакции на ставку.
2. Фьючерсы вглубь: системный анализ базового актива (отдельные разборы через
   macro/market-аналитиков), оценка справедливости контанго/бэквордации, арбитраж,
   парные стратегии. Сделать читаемо (не простыня внизу карточки).
3. Новые классы по протоколу: валюта и металлы — ОТДЕЛЬНЫЙ класс (решение
   владельца); опционы — разрешены; фонды углубить (состав/ошибка слежения/
   дублирование портфеля).
4. Обозреватель рынка (5 под-блоков — см. status.md).
5. Ревизия карточек (fact-checker + читаемость; не пересобирать вкладки).
6. Скрининг (поверх company_metrics).
7. Лендинг (глубокий, маркетинговая интонация).

## 🛡️ ЗАПОВЕДНИК — не трогать без согласия владельца
Новое — делай свободно. Тактику в существующем — правь. НО НЕ переделывай и НЕ
сноси без согласия (даже если кажется лучше): дизайн-систему (бежевый фон+белые

(справедливая цена, 3 уровня достоверности, гибрид-бета, ставка ОФЗ); архитектуру
(стек, «числа в БД / текст в файлах», деплой git push); готовые блоки — вкладки
карточки (Обзор/Бизнес-модель/Финансы/Управление/Рынки/Макро/Гео) и аналитику
портфеля (метрики, корреляции, графики) НЕ удалять/не пересобирать; необратимое
(удаление данных, массовая перезапись) — всегда с согласия. Хочешь изменить
заповедное — доделай остальное и ПРЕДЛОЖИ владельцу в финале, не делай молча.
Крупные изменения — отдельными осмысленными коммитами (откат через git).

## ⚙️ Технические правила (кратко; подробности в playbook)
- Backend-фича: модель → схема → сервис → эндпоинт → тест.
- Схема БД — только Alembic-миграции. Массовая закачка — скриптом.
- Секреты/ключи — только `.env`, НИКОГДА не хардкодить.
- Простые решения вместо «на вырост». Три уровня достоверности (факт/оценка/
  суждение) во всей аналитике. Без рекомендаций «купить/продать».
- Запросы к MOEX ISS — последовательно с паузами, батчами (иначе rate limit).
- Если сессия упала на «rate limit (not your usage limit)» — это повисшая
  сессия/плотный поток, НЕ лимит аккаунта: начни свежую сессию, работай с паузами.
- Объясняй владельцу как НЕ-технарю: видимый эффект → причина → деталь.
