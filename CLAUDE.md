Продукт — инвестиционно-аналитическая платформа для самостоятельного частного инвестора.
Пользовательская проблема:частному инвестору трудно самостоятельно учитывать и постоянно обновлять всю совокупность факторов — от финансов компаний и отрасли до макроэкономики, политики и рыночного фона. Из-за этого решения часто принимаются при неполной картине.

## АРХИТЕКТУРА АНАЛИТИЧЕСКОЙ ПЛАТФОРМЫ

### Три ключевых модуля (из бизнес-требований):

#### Модуль 1: РАЗБОР КОМПАНИИ И ИДЕИ (карточка компании)
Модели и эндпоинты:
- Company (id, ticker, name, sector, description)
- CompanyAnalysis (company_id, overview, arguments_for, arguments_against, risks, fair_price)
- Quote (company_id, date, price, change_day, change_week, change_month)

Эндпоинты:
- GET /api/companies — список компаний по секторам
- GET /api/companies/{ticker} — карточка с быстрым разбором
- GET /api/companies/{ticker}/deep-analysis — полный разбор
- GET /api/companies/{ticker}/stress-test — сценарии

#### Модуль 2: ДИАГНОСТИКА ПОРТФЕЛЯ
Модели и эндпоинты:
- Portfolio (user_id, name, created_at)
- PortfolioPosition (portfolio_id, company_id, quantity, entry_price)
- PortfolioAnalysis (portfolio_id, total_return, risk, diversification, correlations)
- StressScenario (portfolio_id, scenario_name, impact)

Эндпоинты:
- POST /api/portfolios — создать портфель
- POST /api/portfolios/{id}/analyze — анализ портфеля
- GET /api/portfolios/{id}/diagnostics — диагноз (риски, концентрация)
- POST /api/portfolios/{id}/stress-test — стресс-тесты

#### Модуль 3: ОБОЗРЕВАТЕЛЬ РЫНКА (краткий и глубокий)
Модели и эндпоинты:
- MarketUpdate (date, title, description, impact_sectors, importance)
- MarketOverview (date, summary_express, summary_detailed, summary_deep)

Эндпоинты:
- GET /api/market/overview — экспресс-обзор
- GET /api/market/analysis — подробный анализ
- GET /api/market/events — события дня/недели

### Ключевые принципы реализации:
1. Каждая модель = таблица в БД
2. Для каждой модели: модель SQLAlchemy → Pydantic-схема → сервис → эндпоинт → тесты
3. CompanyAnalysis и PortfolioAnalysis — это результаты AI-анализа (потом будет через Claude API)
4. Все числовые результаты должны быть объяснимы (не просто число, а + логика)

### Очерёдность разработки (Часть 5):
1. Company + Quote + CompanyAnalysis (модуль "разбор компании")
2. Portfolio + PortfolioPosition (модуль "портфель")
3. MarketUpdate + MarketOverview (модуль "обозреватель")
