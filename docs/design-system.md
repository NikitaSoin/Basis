# Basis — Design System (единый источник правды)

> **Статус:** канон. Этот документ описывает дизайн-систему, которая **реально работает на бою** —
> **NEO (Basis Neo-Institutional Research System)**. Старая «классическая» система — легаси,
> живёт только за флагом `?classic=1` (см. Приложение). При просьбе «переделай экран» эталон
> один — этот документ.
>
> **Правило заповедника:** дизайн-систему не пересобираем без согласия владельца. Этот файл —
> её ФИКСАЦИЯ (документация), а не пересборка. Любые значения берём из кода
> (`src/styles/tokens.css`, `src/company/neo.jsx`, `index.html`), хардкод hex запрещён.

## 1. Как устроена тема (механизм алиасинга)

Всё приложение оборачивается в класс **`.cc-root`** (на корне `app-layout`, когда NEO активен).
Внутри `.cc-root`:
1. Объявлены NEO-токены `--cc-*` (своя пара значений для light и dark).
2. **Классические имена токенов переопределены (алиас) на `--cc-*`** — поэтому весь существующий
   UI, написанный на старых токенах (`--text-primary`, `--bg-elevated`, `--accent`…), перекрашивается
   в NEO-палитру **без правки кода компонентов**.

Тема переключается классом `.dark` / `.light` на корне (как и раньше). NEO-значения для dark
живут в `.dark .cc-root`.

**Правило:** всегда `var(--token)` или Tailwind-класс (`tw-text-[var(--cc-ink)]`,
`tw-bg-[var(--cc-panel)]`). **Хардкод hex запрещён.**

## 2. Типографика

Шрифты (подключены в `index.html`):
- **Source Serif 4** (`--cc-serif`) — заголовки и редакторская проза (hero-имя, H2/H3, лиды, инсайты, тезисы).
- **JetBrains Mono** (`--cc-mono`) — все числа, тикеры, дельты, проценты, мультипликаторы; `font-variant-numeric: tabular-nums lining-nums`.
- **Inter** (`--cc-sans`) — UI: навигация, кнопки, чипы, eyebrow-лейблы.

**Правило serif:** serif только для редакторской авторитетности (имена/заголовки/проза/инсайты/сценарии).
**Никогда** serif для: eyebrow/лейблов, навигации, кнопок, чипов, чисел (это sans или mono).

Шкала (из спеки карточки; применять единообразно):

| Роль | Семейство | Размер / вес | Трекинг · LH |
|---|---|---|---|
| H1 hero-имя | serif | 50 / 600 | −0.018em · 1.02 |
| H2 заголовок секции | serif | 27 / 600 | −0.015em · 1.1 |
| H3 заголовок плиты | serif | 19 / 600 | −0.01em · 1.3 |
| Лид/тезис | serif | 21 / 400 | 1.5 |
| Проза (чтение) | serif | 16.5 / 400 | 1.72 (≤66ch) |
| Body / UI | sans | 14 / 400–500 | 1.5 |
| Eyebrow/лейбл | sans | 11 / 600 | 0.14em · UPPERCASE |
| Цена (hero) | mono | 52 / 500 | −0.02em |
| Значение метрики | mono | 30 / 500 | −0.02em |
| Таблица/цифры | mono | 13–14 | tabular |

Классы-помощники в `.cc-root`: `.cc-serif`, `.cc-num` (mono+tabular), `.cc-eyebrow`.
Заголовки `h1–h4` внутри `.cc-root` автоматически serif.

ru-RU формат чисел: запятая-десятичный, узкий nbsp в разрядах, nbsp перед единицей.

## 3. Сетка · отступы · радиусы · elevation · анимация

- **Сетка 8pt** (под-сетка 4pt): 2,4,6,8,12,16,20,24,32,40,48,64,80,96.
- **Контент:** макс. ширина карточки `1340px`, гаттер 32px (18px ≤720). Основной грид —
  `minmax(0,1fr) + 332px` правый rail, gap 30px, `align-items:start`; rail sticky. Ритм между плитами 22px.
- **Радиусы:** `--cc-r-lg:14px` (панели) · `--cc-r:12px` (метрики) · `--cc-r-sm:8px` (чипы/кнопки) · pill 9999. (Классические алиасы: `--radius-xs..xl` 4/6/8/12/16.)
- **Elevation:** light — мягкая тень (`--shadow-sm/md/lg/xl`); dark — слой + 1px-граница (`--cc-line`), **без drop-теней и без glow**.
- **Focus-ring:** `0 0 0 3px var(--cc-accent-soft)` + 1px обводка accent — на КАЖДОМ интерактиве.
- **Анимация:** токены `--motion-fast 150` / `--motion-base 250` / `--motion-appear 300` / `--motion-slow 320`; easing `cubic-bezier(0.16,1,0.3,1)`. Hover: подъём/тинт (scale ≤1.015, без bounce). Появление: opacity + y 4px, стаггер ~30мс. **`prefers-reduced-motion` гейтит всё.**
- **ЗАПРЕЩЕНО:** конфетти, elastic, параллакс, scroll-jacking, count-up при ФОНОВОМ апдейте, любое «празднование» изменения цены. Это продукт о доверии, не торговый терминал.

## 4. Эпистемический слой — ОБЯЗАТЕЛЕН (ключевая черта Basis)

Любое аналитическое утверждение несёт **метку достоверности**. Это смысловой слой, не декор.

**CERT** (реализован в `App.js`, используется в renderGeo/renderMacro/renderMarket):

| Уровень | Токен цвета | Лейбл | Смысл |
|---|---|---|---|
| `fact` | `--success` | факт | факт с источником |
| `estimate` | `--warning` | оценка | оценка-ориентир (данные разнятся/устарели) |
| `model` | `--accent` | модель | наша оценка по факторам |
| `logic` | (accent) | механизм | системная логика/теория |
| `judgement` | (accent/violet) | суждение | наше суждение, не предсказание |

**FEJ-тег** (NEO, `src/company/neo.jsx` → `FEJTag`): `fact` (ink-2) · `estimate` (info) ·
`judgment` (accent) · `scenario` (violet). Значение тега инвариантно к теме; меняются только токены.

**Правило:** у любой цифры/вывода/«как доходит до компании» — тег достоверности. Цепочка
передачи (Фактор → Канал → Влияние на бизнес → P&L/FCF → Инвест-вывод) уже реализована в
renderMacro/renderGeo — использовать её как эталон механики.

## 5. Паттерн «4 слоя чтения» (правило для ВСЕХ экранов)

Каждый экран отвечает в одной грамматике, сверху вниз:

1. **Что я смотрю** — идентичность/контекст (кто это, какой инструмент/раздел).
2. **Сигнал** — одно главное число/вердикт, крупно (mono/serif), сразу читается.
3. **Доказательство + надёжность** — данные с эпистемическими тегами (CERT/FEJ), таблицы/графики.
4. **Действие** — что с этим делать (decision-support, без «купить/продать»).

**Принцип: слой ВЕРДИКТА поверх слоя ДАННЫХ.** Голая таблица/сетка без интерпретации
(«что это значит») — **не готовый экран**. Эталон — вкладка «Финансы и оценка» (`renderFinancials`):
hero справедливой цены → методы с выкладкой → мультипликаторы → вывод.

## 6. Цвет — только в данных

- **Хром монохромный** (фон/навигация/кнопки): нейтраль + кобальт-accent для интерактива. Никаких декоративных цветов в хроме.
- **Семантика дельт:** `--success`/`--danger` **только** в паре с глифом ▲/▼/▬, никогда как украшение.
- **Категориальная палитра** `--cat-1..8` (Okabe-Ito, colorblind-safe) — **только в визуализациях данных** (серии графиков, секторные бейджи, treemap, полосы метрик). Дозировка: 2–3 цветовых акцента на блок, остальное нейтрально. Не «радуга», не «мёртвая серость».

## 8. Доступность (не обсуждается)

- WCAG 2.1 **AA** минимум (цель AAA для текста). Токены подобраны под контраст в обеих темах.
- У каждой прибыли/убытка — глиф **▲/▼/▬**, никогда только цвет.
- **Focus-кольца** на каждом интерактиве (`--cc-accent-soft`).
- Тач-зоны ≥ 44×44 на мобильном (≥32 десктоп).
- `prefers-reduced-motion` учитывается везде.
- Респонсив: ≥1080 — две колонки; ≤1080 — rail над контентом, метрики 3-в-ряд; ≤720 — одна колонка.
