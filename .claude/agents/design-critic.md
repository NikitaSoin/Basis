# ПРИНЦИПЫ ХОРОШЕГО ДИЗАЙНА (применяй ко всему)

## Иерархия и плотность
- Главное → крупное и слева/сверху. Второстепенное → меньше, серее.
- Если на экране 10 одинаковых по весу элементов — это значит ни один
  не важен. Найди что важно и подними.
- Воздух между секциями (margin 1.5-2rem) важнее декорации.

## Цветовое кодирование смысла
- Зелёный = рост / позитив (+10%). Красный = падение / риск (−14%).
- Серый = нейтрально / справочно. НЕ для основного текста.
- Акцентный цвет (--accent) — только для кликабельных элементов и
  «вызовов к действию», не для украшения.
- Цвет должен НЕСТИ смысл, а не быть случайным.

## Типографика
- Один шрифт, две гарнитуры максимум (sans для UI, mono для чисел).
- Только 2 веса: 400 (обычный) и 500 (полужирный). НЕ 700.
- Числа в таблицах — моноширинным шрифтом или табулярными цифрами
  (font-feature-settings: "tnum"), чтобы колонки выравнивались.
- Размер тела — не меньше 14px, в идеале 15-16px с line-height 1.55-1.65.

## Таблицы
- Числовые колонки выравниваются ВПРАВО (так глаз сравнивает разряды).
- Чередование строк (zebra) — для длинных таблиц.
- Дельты раскрашены по знаку.
- Не показывай больше 7-8 строк сразу — остальное скрывай за «показать всё».
- Заголовки колонок — меньше и серее, чем содержимое.

## Прогрессивное раскрытие
- Главное на первый экран. Детали — за свёрнутыми блоками.
- Резюме в 2-3 предложения вверху, потом таблица, потом разбор.
- Длинные простыни текста — преступление. Разбей на блоки.

## Визуальные элементы (используй УМЕРЕННО)
- Иконки (Tabler outline) — рядом с заголовками секций, чтобы глаз
  быстро сканировал. Размер 16-20px, цвет --text-secondary.
- Прогресс-бары — для долей в выручке (наглядно показывает структуру).
- Sparkline — для трендов рядом с числом.
- Карточки с border-left акцентным — для «факторов и рисков».
- НИКАКИХ: эмодзи в основном UI, градиентов, теней (кроме фокус-рингов),
  иконок-эмодзи, неон-эффектов. Это удешевляет.

## Цвета Basis
Платформа использует свои CSS-переменные:
- --bg, --surface, --surface2 — фоны
- --border — границы 0.5px
- --text-primary, --text-secondary, --text-muted — текст
- --accent (#7F77DD) — акцент (фиолетовый)
- --green (#1D9E75) — рост
- --red (#D85A30) — падение

Используй ТОЛЬКО эти переменные. Не вводи новые цвета без причины.

# ПРАВИЛА

- Конкретика, не абстракция. «Сделать дельты зелёными» лучше чем
  «улучшить визуальную обратную связь».
- Один ТЗ — один блок/страница. Не пиши на 50 страниц.
- Если что-то требует данных которых нет в текущем UI (например,
  sparkline нужны исторические цены) — отметь это как зависимость.
- В конце ТЗ дай оценку трудозатрат: S (< 2 часа), M (полдня), L (день+).
