# Корпоративное управление — ПАО «Мечел» (MTLR / MTLRP) — синтез

*Данные на 25.06.2026. Качество данных: medium. Балл управления: 1,9 из 5 — флаг **weak** (норма сектора металлургии = 3,0).*

## Вердикт
**Слабое управление (weak).** Структура контроля раскрыта прозрачно (семья Зюзина ~51,5% голосов, конечный бенефициар известен, МСФО публикуется регулярно), но *поведение* устойчиво работает против миноритария: денежные потоки прибыльных дочек переливаются «вверх» к закредитованной материнской структуре, дивиденды не платятся ни по одному классу, а уставная защита префа игнорировалась в прибыльные годы. Управление обслуживает кредиторов и контролирующего собственника, не розничного держателя.

## Почему такой балл (ключевые факты)
*(факт)* Два фактора из восьми получили балл 1 (красный флаг) — оба активны как override-триггеры методички:
- **related_party_flows = 1, dividend_stability = 1.** Системный вывод FCF через связанные стороны (по «Уралкузу» 2024 ~36,1 из ~47,4 млрд руб займов — самому «Мечелу»; условия исторически оспаривались как нерыночные, частично поддержаны судом — Арсагера, 2013) И нарушение зафиксированного уставом правила 20% ЧП по префам в три прибыльных года подряд (FY2021–2023, прибыль 80,6 / 62,9 / 22,3 млрд руб — выплачено ноль).

*(оценка)* Остальные факторы ниже нормы (2) либо на нижней границе (3): защита миноритариев, разрыв голос/капитал, контроллер-экстракция, независимость совета (реальная низкая при номинальном большинстве независимых — совет возглавляет сам собственник), прозрачность (SDN-санкции с 02.2024, кипрский квазиказначейский периметр Skyblock). Контроллер-экстракция не опущена до 1: поток сейчас идёт не в карман собственнику, а внешним госбанкам-кредиторам.

*(вывод для миноритария)* Денежного возврата нет и не предвидится; уставное обещание по префу не является гарантией; риск экспроприации через переток к материнской структуре — структурный, а не разовый.

## Связь с оценкой (assessment)
Балл 1,9 + два активных красных флага помещают компанию в **override-зону** маппинга (контур «б», выпуклая кривая → премия к ставке у потолка). Конкретное число governance-премии (`premium_to_wacc_pp`) считает финанс-слой по `config/governance_mapping.json` — здесь оставлено `null`.

**Важно, чтобы не задвоить эффект:** долговая ловушка (отрицательный капитал −179 млрд, чистый долг ~246 млрд, поток на кредиторов) уже отражена в `financials.json` напрямую — equity по DCF и EV/EBITDA на нормализованной базе отрицательна, цена = опцион на восстановление. Governance-премия должна добавлять *именно управленческий* риск (tunneling + неисполнение прав префа), а не повторно штрафовать за долг.
