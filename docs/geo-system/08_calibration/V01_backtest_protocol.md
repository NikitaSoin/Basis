# V01. Протокол бэктест-калибровки

**Слой:** методология. Веса, пороги и типовые амплитуды берутся из истории, а не из интуиции (Принцип 6). Два режима: **ретроспективный бэктест** (запускается при внедрении, повторяется при каждой мажорной ревизии базы) и **проспективный лог** (копится с первого боевого барометра через журнал B03 §6 и строки watchlist — со временем становится главным материалом).

---

## 1. Что калибруется

(а) Стартовые веса свёртки G-композита — в B01 они сознательно не зашиты (профиль важнее свёртки); до первого бэктеста агент использует равные веса с двойным весом G1, G3, G5, G12 как приор [ОЦЕНКА-приор, помечается в каждом отчёте до калибровки]. (б) Типовые амплитуды реакций (B02: диапазоны S1–S4; C06: величины премий/haircut'ов). (в) Пороги триггеров B03 (какой шаг оправдан событием какого класса). (г) Коэффициент затухания пис-беты (K20→K24→K25). (д) Качество причинной атрибуции C08 (доля «гео» не завышена).

## 3. Freeze-процедура (защита от ретроспективного знания)

Для каждого эпизода: (1) фиксируется дата T (для №6 и №8 — дата события; для санкционных — канун удара). (2) Составляется досье строго из источников, датированных <T; **каждый факт в досье несёт дату публикации**; отдельный контролёрский проход ищет анахронизмы (утечки послезнания — главный риск процедуры, для LLM особенно: модель знает историю; freeze снижает, не устраняет — это честно фиксируется в confidence итогов). (3) Агент по промптам и базе выставляет: G-вектор на T; GRE-профили целевых бумаг; сценарные вероятности; **направленные условные прогнозы** («если событие X происходит в течение N месяцев → ожидаю: знак и диапазон реакции корзины из 6–10 бумаг, ранжирование внутри корзины, вторичные эффекты»). (4) Вскрытие: сравнение с реализацией по метрикам §4. (5) Разбор расхождений письменно: ошибка данных / ошибка шкалы / ошибка механики / непрогнозируемое честно (найтова компонента — не штрафуется, если была помечена ex ante).

## 4. Метрики и пороги приёмки

**Событийный слой:** Brier score по бинарным строкам (реализуется/нет) против наивной базы (климатологическая частота); требование: лучше базы на выборке в целом.
**Направленный слой:** доля верных знаков реакции ≥75% по эпизодам; ранговая корреляция Спирмена прогнозного и фактического упорядочения бумаг в корзине: медианная ρ ≥ 0,5.
**Калибровка величин:** заявленные интервалы покрывают факт в ≥70% случаев И не «резиновые» (средняя ширина интервала фиксируется и не растёт между итерациями без обоснования — иначе калибровка куплена бессодержательностью).
**Атрибуционный слой (на №3, №5 + резерв K28):** сумма долей каналов ≤100%; доля «прямого гео» не выше контрольной декомпозиции, построенной через глобальных пиров (метод C08 §3); специально проверяется НЕсрабатывание — эпизод №8: система, выдавшая X5 высокий гео-риск после снятия E9, проваливает тест деформации рамки независимо от остальных результатов.
**Разграничительный слой (№3):** корректная маркировка «фискальная экспроприация → ведёт ИНСТ, ГЕО даёт фон» — проверка таблицы владельцев A02 в действии.

## 5. Правила калибровки весов

Стартовые значения = приоры (§1). Корректировка ТОЛЬКО по итогам полного прогона, с записью: «параметр X: было → стало; основание — эпизоды №…; величина сдвига». Ограничение анти-переобучения: ни один вес/порог не меняется более чем на 20% за одну итерацию; изменения, опирающиеся на единственный эпизод, помечаются «слабое основание» и подлежат первоочередной проверке проспективным логом. Результат фиксируется в kb_version (git-commit в каждом отчёте агента — README §7); действующая версия весов цитируется в шапке каждого барометра.

## 6. Периодичность

Полный ретроспективный прогон: при внедрении; далее — при каждой мажорной ревизии базы (новые якоря шкал, новые кейсы в эталонной выборке). Лёгкая сверка по проспективному логу: ежеквартально (совмещена с B03 §6: точность триггеров, лаги, пропуски). Внеплановая: после любого события, сдвинувшего ≥2 субиндексов на ≥0,5 (проверка: предсказала ли действующая версия структуру каскада).

## 7. Формат отчёта о бэктесте

Шапка (kb_version, дата, состав выборки) → таблица «эпизод × метрики» → разбор каждого расхождения (классификация ошибки из §3.5) → решения по параметрам (было/стало/основание) → нерешённые вопросы в очередь ревизии базы → приложение: полные freeze-досье и выходы агента (воспроизводимость). Отчёт — версионируемый артефакт; следующий бэктест обязан сослаться на предыдущий и показать динамику метрик (Принцип 9: опись, ничего не выбрасывается молча).

## 8. Ограничения — честно

N=8 — микровыборка: статистическая значимость недостижима, бэктест здесь — детектор грубых дефектов механики и деформаций рамки, а не оптимизатор. Survivorship: эталоны — громкие эпизоды; «тихие» адаптации недопредставлены → пороги реакций смещены вверх; поправка: направленные прогнозы формулируются и для «скучных» периодов между эпизодами (нулевая реакция — тоже проверяемый прогноз). Двойного слепого нет: и составитель досье, и агент «знают историю» — freeze-протокол и контролёрский проход снижают утечку, остаточная — в confidence. Из этого следует главный вывод протокола: **ретроспективный бэктест даёт лицензию на запуск, но настоящая калибровка начинается с проспективного лога** — поэтому журнал B03 §6 и строки watchlist обязательны с первого дня, без исключений.
