## Главный вопрос (JTBD)
> **«Что на самом деле произойдёт с моими деньгами, если я возьму этот опцион, —
> и оправдан ли его риск тем, что я хочу получить?»**

Иерархия ответа (факт → логика → оценка):
1. **Что это и выплата** (факт+логика): call/put, страйк, экспирация, базовый
   актив (фьючерс), профиль выплаты, точка безубытка, макс. убыток (=100% премии
   для покупателя; неограниченный для продавца).
2. **Сколько стоит и почему** (факт+логика): премия = внутренняя + временная
   стоимость; сколько «воздуха» и как быстро его съедает тета.
3. **Дорого/дёшево сейчас** (оценка): подразумеваемая волатильность (IV) vs её
   история / реализованная волатильность БА.
4. **Греки человеческим языком** (логика): дельта (≈ движение опциона / грубо
   вероятность исполнения), тета (потеря в день от времени), вега (боль от
   изменения волатильности) — 3-4 фразы, не таблица букв.
5. **Связь с портфелем** (отложено): сколько put-ов хеджируют позицию, цена
   страховки в % портфеля.

## Позиционирование (критично — «не казино»)
Опцион опаснее фьючерса: тета-распад (теряешь, даже будучи правым по направлению),
вега, продажа опциона = неограниченный убыток. Подача:
- Явный риск-баннер сильнее, чем у фьючерса: «инструмент, который чаще всего
  работает против новичка — вот как он устроен».
- Центр экрана — профиль УБЫТКА и тета, а не прибыль.
- НЕ показывать: сигналы «купить опцион / IV вырастет», топ по доходности,
  скринер «иксов», готовые стратегии с «сделай так». Без поощрения частой торговли.

## Реализация — Вариант C (рекомендован): класс с УРЕЗАННОЙ витриной
НЕ вся доска опционов (это терминал/казино), а: к базовому активу привязаны
несколько страйков около денег (±несколько от центрального) + ближайшие 1-2
экспирации; клик по опциону → разбор по иерархии. Плюс образовательный калькулятор
выплаты/хеджа. Полная доска (A) = терминал (отверг); чистый калькулятор (B) =
недоинтегрирован.

## Данные (MOEX ISS, публично)
- Справочник/цепочка: `/iss/engines/futures/markets/options/securities` — список,
  STRIKE, тип, центральный страйк, расчётная цена, даты торга/исполнения.
- Греки/IV/теор.цена/внутр.+врем. стоимость: API опционного калькулятора MOEX
  (`iss.beta.moex.com/iss/apps/option-calc/v1`, публичный) ИЛИ считать самим
  (Блэк-Шоулз от фьючерса + IV из кривой/рыночной цены) — в духе Basis (своя
  методика, как гибрид-бета). beta-API проверить на стабильность перед продом.

## Уровни достоверности
- факт: премия/расчётная цена/страйк/даты.
- логика (детерминированная): внутренняя стоимость, безубыток, профиль выплаты,
  макс. убыток.
- оценка (модельная, помечать): греки, IV, теор. цена (модель B-S/Башелье; у
  неликвидных/дальних страйков шумны — дисклеймить).
