# A02. Протокол взаимодействия: ГЕО-МАКРО ↔ ГЕО-КОМПАНИЯ ↔ ИНСТ-система ↔ Red-team

## 1. Граница между ГЕО- и ИНСТ-системами (правило единственного владельца)

Каждый фактор риска имеет ровно **одну систему-владельца**, которая его оценивает и транслирует в дисконт. Вторая система **импортирует** оценку и ссылается на неё, но не пере-оценивает и не добавляет в свой дисконт. Это анти-двойной счёт уровня систем — без него один и тот же риск (например, «государство может отнять») войдёт в цену дважды: через IRI и через GRE.

| Фактор | Владелец | Обоснование и режим импорта |
|---|---|---|
| Санкции (прямые, вторичные), экспортконтроль | **ГЕО** | Источник — вне периметра РФ. ИНСТ импортирует как экзогенный шок для своих субиндексов. |
| Война: интенсивность, мобилизация, удары | **ГЕО** | — |
| Переговоры, сценарии мира | **ГЕО** | — |
| Экспортные каналы, дисконты, флот, платежи | **ГЕО** | — |
| Замороженные активы, конфискации ЗА рубежом | **ГЕО** | — |
| Контрсанкционный режим РФ (указы об особом порядке сделок, «выходной налог», обязательная продажа выручки, внешнее управление иностранными долями) | **Совместно: триггер — ГЕО, механика — ИНСТ** | Возникает как ответ на внешнее давление (ГЕО прогнозирует появление/снятие через сценарии), но исполняется внутренними институтами (ИНСТ оценивает жёсткость применения, избирательность, риск для конкретного эмитента). В дисконт эмитента входит один раз — через ИНСТ-канал; ГЕО лишь задаёт сценарную вероятность смягчения/ужесточения режима. |
| Фискальные изъятия военного времени (windfall, «нашлёпки» НДПИ, рост НДС) | **Совместно: масштаб потребности — ГЕО (G12), кто заплатит — ИНСТ** | ГЕО оценивает размер фискального разрыва войны; ИНСТ — механику выбора «доноров» (кланы, переговорная сила). В FCF эмитента входит один раз, атрибуция указывается двойная. |
| Права собственности, суды, силовое давление, кланы | **ИНСТ** | ГЕО импортирует итоговую оценку внутренней стабильности; не строит свою. |
| Внутренняя политическая устойчивость, транзит власти | **ИНСТ** | ГЕО учитывает только интерфейсный член: «военный стресс → внутренняя устойчивость → смена внешнего курса» — как триггеры в сценариях B02, без собственного балла. |
| Ключевая ставка, инфляция, курс как макро-величины | **Ничей дисконт** | Входят в оценку через безрисковую ставку и рыночные параметры; обе системы объясняют их драйверы (ГЕО — военно-фискальный вклад, ИНСТ — институциональный), но НЕ добавляют повторно в компанейские премии. См. C06 §3. |
| Навес нерезидентов, счета «С», доступность бумаги иностранному капиталу | **ГЕО** | Источник — санкционный/переговорный статус (снятие навеса зависит от сценария урегулирования, не от внутренних институтов РФ). ИНСТ импортирует как контекст ликвидности, не оценивает заново. |
| Платёжно-курсовые трансакционные издержки экспортёра (страховка теневого флота, задержки расчётов, вторичные санкции на контрагентов-посредников) | **ГЕО** | Часть контура «Экспортные каналы, дисконты, флот, платежи» (см. выше) — уточнено отдельной строкой 2026-07-12 после того, как red-team пилотного батча (ROSN/LKOH/GAZP) нашёл, что ИНСТ систематически включал этот контур в свою WACC-премию как «институциональный» риск, хотя источник риска — внешний (санкции), не внутренний (кланы/суды). |

Если фактор не найден в таблице — агент обязан явно предложить владельца и пометить решение [ОЦЕНКА], а не молча оценить сам.

**🔴 Обязанность ИНСТ (добавлено 2026-07-12, после находки red-team на пилотном батче geo-company-analyst):**
ИНСТ-агент (institutional-macro-analyst / institutional-company-analyst) ОБЯЗАН перед расчётом WACC-премии/FCF-haircut/мультипл-дисконта пройти по этой таблице и исключить из СВОЕГО расчёта любой компонент, чей источник — санкции, экспортные каналы, платежи/логистика теневого флота, навес нерезидентов, счета «С» — даже если формально он проявляется как «институциональный» риск (например, через субиндексы вида «санкционный/внешний контур», «трансакционный режим платежей»). Такие субиндексы допустимо ДЕРЖАТЬ в барометре ИНСТ для внутренней калибровки, но их величина НЕ должна попадать в итоговые WACC/FCF/multiple числа, которые ИНСТ публикует для сборки карточки — она принадлежит ГЕО. Нарушение этого правила было системным на пилотном батче (2026-07: ROSN, LKOH, GAZP из 4 проверенных red-team) и приводило к гарантированному двойному счёту при механической сборке.

## 2. Handoff ГЕО-МАКРО → ГЕО-КОМПАНИЯ

Финальный блок каждого отчёта МАКРО (структура фиксирована):

```
=== HANDOFF GEO-MACRO ===
date: YYYY-MM-DD
kb_version: <commit-hash>
G_scores: {G1: x.x (Δ±), ... G13: x.x (Δ±)}
scenarios: {S1_breakthrough: p% (Δ), S2_ceasefire: p% (Δ), S3_attrition: p% (Δ), S4_escalation: p% (Δ)}
scenario_confidence: L/M/H + одно предложение основания
sector_flags: [сектор: направление/причина — только изменившиеся]
watchlist_30d: [3–7 наблюдаемых событий-сигналов с ожидаемым эффектом на G/сценарии]
stale_after: <дата или триггер, после которого барометр считать устаревшим>
===
```

Правила для КОМПАНИИ: (1) процитировать шапку handoff в своём отчёте; (2) при барометре старше `stale_after` или при срабатывании major-триггера из watchlist — остановиться и запросить обновление МАКРО; (3) запрещено самостоятельно менять G-баллы; допустимо аргументированно оспорить — через эскалацию (см. §4).

## 3. Обратный поток (bottom-up валидация)

Ежеквартально или по накоплении ≥5 свежих отчётов КОМПАНИИ: свод медиан и хвостов E-баллов по покрытию → сверка с G. Формализованные проверки: медиана E4 (платёжные каналы) против G4; медиана E3 (логистика) против G5; частота срабатывания E1-триггеров против G3. Расхождение ≥1,0 балла в устойчивую сторону = алерт «барометр отстаёт от микроданных», разбирается человеком с обязательным пост-мортемом (кто был прав и почему).

## 4. Разногласия и эскалация

Агенты не «договариваются» между собой молча. Разногласие (КОМПАНИЯ считает G-балл неверным; ИНСТ и ГЕО претендуют на один фактор; red-team против автора) фиксируется в отчёте отдельным разделом «Разногласия», с позициями сторон и данными каждой, и выносится человеку. Запрещено: усреднять позиции без решения, скрывать разногласие выбором одной цифры.

## 5. Red-team: место в конвейере

Red-team получает готовый черновик отчёта КОМПАНИИ (или МАКРО при крупных пересмотрах) и базу. Возвращает: список замечаний по чек-листу (двойной счёт внутри отчёта; двойной счёт ГЕО↔ИНСТ; двойной счёт с рыночной ценой; эндогенность; сверхатрибуция гео-рамке; survivorship bias кейсов; ложная точность; nirvana fallacy; несоответствие якорям). Автор обязан на каждое замечание ответить: принято (что изменил) / отклонено (почему). Диалог прикладывается к финальному отчёту. Отчёт без red-team-приложения считается черновиком.

## 6. Совместный отчёт по эмитенту (ГЕО + ИНСТ)

Итоговая инвестиционная картина по эмитенту собирается из двух отчётов по правилу владельца (§1): итоговая таблица «фактор → владелец → канал трансляции → величина», где каждый фактор встречается один раз. Сборку выполняет ГЕО-КОМПАНИЯ (как последний в конвейере) либо человек; при сборке обязательна проверка пересечений по списку совместных факторов из §1.
