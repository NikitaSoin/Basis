# Сравнение структур блока «Бизнес-модель»: lean (новая) vs rich (старая)

Дата: 2026-05-31
Автор: агент-диспетчер (по итогам A/B-эксперимента на LKOH, GAZP, YDEX)

## Что сравнивали

У трёх компаний (ЛУКОЙЛ, Газпром, Яндекс) было две версии блока:

- **rich** — старая «богатая» 10-секционная структура (`business_model_legacy.md`):
  Краткое резюме → 1. Суть → 2. Структура выручки → 3. Добыча/Производство и
  ресурсная база → 4. География → 5. Структура затрат → 6. Конкурентная позиция
  (рвы) → 7. Регуляторная среда → 8. Макрочувствительность → 9. Финансовые метрики
  → 10. Ключевые метрики для мониторинга.
- **lean** — новая структура по эталону ROSN (`business_model.md`), которую получили
  все остальные ~260 компаний: Первый экран (Суть + мини-P&L + факторы/риски) →
  Экономика и детали (как формируется выручка, расходы до/после EBITDA, сегменты,
  цепочка стоимости, география, клиенты).

По каждой версии прогнали субагента-персону (целевой пользователь Basis).

## Оценки персоны (1–5), файлы persona_feedback*.md

| Компания | lean (новая) | rich (старая, legacy) |
|---|---|---|
| LKOH | Понятность 4 / Достоверность 4 / Полнота 4 / Удобство 4 (**4,0**) | 4 / 4 / 3 / 3 (**3,5**) |
| GAZP | 4 / 4 / 3 / 4 (**3,75**) | 3 / 4 / 3 / 3 (**3,25**) |
| YDEX | 4 / 3 / 3 / 4 (**3,5**) | 4 / 5 / 4 / 3 (**4,0**) |

Вывод по баллам неоднозначен (lean выигрывает у нефтяников по «удобству», rich
выигрывает у Яндекса по «достоверности/полноте»). Но КАЧЕСТВЕННЫЕ комментарии
персоны дают чёткий сигнал, что именно ценно в каждой версии.

## Что персона ХВАЛИЛА в rich (и чего НЕТ в lean)

1. **Раздел 10 «Ключевые метрики для мониторинга»** — упомянут во всех трёх ревью
   как «золотой стандарт» и «инструмент, а не справочник»: инвестор может открыть
   карточку через месяц и за 2 минуты проверить красные флаги (добыча −5%,
   EBITDA-маржа −20%, долг +X). **Самый востребованный элемент.**
2. **Производственная и ресурсная фактура** (раздел 3): число и мощность НПЗ,
   глубина переработки/индекс Нельсона, объёмы добычи, запасы и их выработанность,
   маршруты экспорта, доли рынка. Для производственных компаний это «мясо», которое
   инвестор хочет видеть и которого в lean-версии нет.
3. **Сегментные EBITDA** (по бизнес-юнитам отдельной строкой) — особенно ценно для
   диверсифицированных компаний (Яндекс: «видна двухскоростность бизнеса»).
4. **Макрочувствительность и регуляторная среда отдельными блоками** — «в России
   критично».

## Что персона ХВАЛИЛА в lean (и чего не хватает в rich)

1. **Связка «выручка → прибыль»**: мини-P&L + «Расходы до EBITDA» / «Расходы после
   EBITDA» прямо отвечают на главный вопрос «почему у компании именно такая
   прибыль». В rich этого мостика нет — по Газпрому персона прямо жаловалась, что
   «логика роста выручки +29% не объяснена», «структура затрат не показана как %
   выручки», «убыток не разложен на курсовые vs EBITDA».
2. **Компактность и читаемость**: rich «перегружен таблицами», «утомляет», «шумит
   микро-продуктами», «на телефоне неудобно».

## РЕКОМЕНДАЦИЯ

**Базой оставить lean-структуру** (её P&L-мостик «выручка → EBITDA → прибыль» —
главное преимущество и ядро смысла блока). Но **вернуть из rich два элемента**:

### Для ВСЕХ компаний — добавить блок «Ключевые метрики для мониторинга»
Короткий чек-лист (5–8 строк) с пороговыми сигналами для регулярной проверки.
Дёшево по объёму, высоко ценится персоной, применимо к любому эмитенту.

### Только для КРУПНЫХ ПРОИЗВОДСТВЕННЫХ компаний — добавить блок «Производство и ресурсная база»
Целевые сектора: нефтегаз, металлургия, электрогенерация, крупное машино- и
химпроизводство. Внутри: производственные мощности (НПЗ/заводы/станции), объёмы
добычи/выпуска, запасы и их выработанность, доли рынка, ключевые активы на карте.
Для сбытов, банков, IT, ритейла, лизинга, девелопмента, страховщиков — НЕ нужен
(там производственной базы нет, lean самодостаточен).

### Где полезны сегментные EBITDA
Для диверсифицированных холдингов (Яндекс, Интер РАО, АФК Система-типа) — давать
EBITDA по сегментам отдельной строкой внутри раздела «Сегменты». Для моно-бизнесов
(сбыты, отдельные НПЗ, региональные сети) — не требуется.

### Чего НЕ переносить из rich
Дублирующие разделы «Макрочувствительность» и «Регуляторная среда» целыми блоками —
по правилам аналитика они относятся к соседним карточкам (Макроэкономика, и т.п.);
достаточно упоминания в «Факторах и рисках». Не плодить микро-метрики и лишние
таблицы — сохранить компактность.

## Практический следующий шаг (когда будет согласовано)
1. Дополнить промпт `business-model-analyst` опциональными блоками «Производство и
   ресурсная база» (флаг для производственных секторов) и «Ключевые метрики для
   мониторинга» (для всех).
2. Прогнать апдейт по крупным производственным эмитентам (нефтегаз, металлургия,
   генерация) — у них эффект максимальный.
3. Legacy-файлы (`business_model_legacy.md` у LKOH/GAZP/YDEX) пока сохранены как
   черновики-образцы богатого формата; удалить после внедрения.
