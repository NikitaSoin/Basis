# План Фазы 4 — движение, логомарк, финальный полиш (только план)

Цель: завершить дизайн-систему дозированным движением и брендом. Принцип прежний —
**доверие, а не торговое приложение**: 2-3 «точки жизни» на экран, ничего не
мельтешит, всё короткое и осмысленное. Раскатка — ПО ОДНОМУ куску, с проверкой
владельца на живом (движение легко переборщить).

## 0. Принципы движения (из конституции «Живость и глубина»)
- Короткое: **150-250 мс**, единый базовый тайминг **~250 мс**. Easing появления —
  `cubic-bezier(0.16, 1, 0.3, 1)`.
- **`prefers-reduced-motion` ОБЯЗАТЕЛЕН** на КАЖДОМ эффекте — откат на мгновенное
  или opacity-only.
- ЗАПРЕЩЕНО: мельтешение, конфетти, bounce/elastic, параллакс, scroll-jacking,
  «празднование» изменения цены, count-up при фоновом обновлении.
- Дозировка: ≤1 медленный декор на раздел; не анимировать всё подряд.

## 1. Motion-токены (новое, в tokens.css) — шаг 4a
Завести и использовать ВЕЗДЕ вместо хардкод-длительностей:
```
--motion-fast:  150ms;   /* hover, micro */
--motion-base:  250ms;   /* появления, count-up, tab-slide — ЕДИНЫЙ */
--motion-slow:  320ms;   /* модалка/страница */
--ease-out:     cubic-bezier(0.16, 1, 0.3, 1);
--motion-decor: 18s;     /* медленный дышащий/orbit-декор */
```
Tailwind: прокинуть как `duration-base`/`ease-out` если удобно. reduced-motion —
глобальный `@media` уже есть, расширить.

## 2. ⛔ ПРАВИЛО COUNT-UP (критично — не повторить баг портфеля)
**Count-up играет РОВНО ОДИН раз при ПЕРВОМ появлении числа на странице. НЕ
переигрывается при: переключении табов, кликах, ре-рендере, фоновом обновлении
(опрос цен 5 c), наведении.** При reduced-motion — сразу финал.

Механизм (УЖЕ применён, не ломать): компонент-обёртка count-up вынесен на
МОДУЛЬНЫЙ уровень (стабильная идентичность, нет ремоунта при ре-рендере) +
`gate`-ref `{played:false}` живёт на уровне СТРАНИЦЫ (CompanyCard/PortfolioView,
переживает переключение табов). `useCountUp(value, duration, gate)`: если
`gate.played` — снап без анимации.

Аудит существующих count-up (ВСЕ уже по правилу — в 4a только сменить тайминг
700→`--motion-base` 250 мс, поведение НЕ трогать):
| Где | Число | Gate-ref | Статус |
|---|---|---|---|
| Обзор | справедливая цена (P/E) | `ovwCountGate` | ✅ gated |
| Финансы | мультипликаторы | `finCountGate` | ✅ gated |
| Корп. управление | free float % | `govCountGate` | ✅ gated |
| Портфель | Стоимость | `valueGate` | ✅ gated |
| Портфель | Индекс /100 | `scoreGate` | ✅ gated |

**Любой НОВЫЙ count-up в Фазе 4 ОБЯЗАН:** (1) компонент на модульном уровне;
(2) gate-ref на уровне страницы; (3) duration = `--motion-base`; (4) reduced-motion
→ финал. Дозировка: count-up ТОЛЬКО на ГЛАВНОМ числе блока/плитки, НЕ на каждом
числе (иначе мельтешение). Кандидаты на добавление (обсудить, не обязательно):
ключевые KPI на карточке (если есть hero-числа без count-up), индексы качества/
доходности/риска (когда появятся). НЕ добавлять на: котировки в списке Рынка
(они фоново обновляются — count-up там запрещён), числа в таблицах (их много).

## 3. Инвентарь движения (дозировано) — шаги 4b-4d
**4b. Появление карточек/секций** (главная «жизнь»): при первом монтировании
страницы/вкладки — `opacity 0→1` + `translateY 4px→0`, стаггер 30 мс между
карточками, `--motion-base`, `--ease-out`. ТОЛЬКО при входе (как count-up, не на
ре-рендере/табах — иначе моргание). reduced-motion → мгновенно. Применить к:
сеткам карточек Обзора (bento), плиткам KPI Финансов, секциям Управления,
карточкам Портфеля, списку Рынка (стаггер по строкам, но аккуратно — много строк →
ограничить первые ~12 или отключить стаггер для длинных списков).

**4c. Микро-взаимодействия (консистентность)**: hover-подъём карточек
(`scale 1.0→1.015` + тень, `--motion-fast`) — свести к ЕДИНОМУ паттерну везде, где
карточка кликабельна; focus-ring уже единый; tab-slide уже есть (свести тайминг к
`--motion-base`); кнопки — hover/active тайминг к токену. Никаких новых эффектов —
только унификация существующих под токены.

**4d. Один «дышащий»/orbit-декор** (приём Fiscal) — ОЧЕНЬ сдержанно, ≤1 на крупный
экран: тонкий медленно-вращающийся/пульсирующий SVG-элемент низкой непрозрачности
в hero лендинга (и, опц., в шапке карточки). `--motion-decor` 18 c, opacity ~0.04-0.08,
чисто фон. reduced-motion → статичный. Это «живость без мельтешения». Обсудить:
нужен ли вообще, или достаточно появлений+count-up.

(Опц. 4x) Переход между маршрутами/вкладками — лёгкий cross-fade (`--motion-base`),
если не мешает. Низкий приоритет.

## 4. Логомарк (отдельный пункт — варианты, владелец выбирает) — шаг 4e
Сейчас лого = иконка `Activity` (заглушка). Нужен знак бренда «Basis» —
инвестиционная аналитика, ценности: доверие, логика, основа/база, снижение
неопределённости. Подготовлю **3-4 концепт-направления** логомарка и покажу на
`/_design` (в обеих темах, на тёмном/светлом, в размере фавикона и в шапке):
- Напр.: монограмма «B» как фундамент/слои; абстракция «опора/базис» (горизонталь-
  основание + восходящий элемент); «призма/грань» (аналитический слой/второе мнение);
  минималистичный знак из сетки/осей (данные).
Владелец выбирает направление → дорабатываю → ставлю в сайдбар + фавикон + (опц.)
лендинг. Палитра — кобальт-акцент + нейтрали, без декоративных цветов.

## Порядок раскатки (по одному, с проверкой владельца)
1. **4a — Motion-токены + единый тайминг + аудит/правило count-up** (250 мс,
   gate везде). Низкий визуальный риск, фундамент. push → reviewer → проверка.
2. **4b — Появление карточек/секций** (opacity+y, стаггер, только при входе).
   Главная «жизнь». push → reviewer → проверка (на мельтешение/повтор при табах!).
3. **4c — Унификация микро-взаимодействий** (hover/focus/tab под токены).
4. **4d — Один orbit-декор** (если владелец захочет). push → reviewer → проверка.
5. **4e — Логомарк** (варианты → выбор → внедрение).
Каждый кусок — отдельный коммит, `CI=true` сборка, reviewer, живая проверка.
НЕ отдавать всё пачкой.

## Definition of Done Фазы 4
- Каждый эффект 150-250 мс, единый `--motion-base`; `prefers-reduced-motion`
  гейтит ВСЁ (проверка с включённым reduced-motion — сайт статичен, числа на финале).
- НИ ОДИН count-up не переигрывается при табах/кликах/фоновом апдейте (проверка:
  войти → число раскрутилось 1 раз → переключать табы/ждать опрос цен → стоит).
- Движение дозировано (2-3 точки на экран), не мельтешит, обе темы, доступность AA.
- Логомарк выбран владельцем, стоит в сайдбаре + фавиконе.
