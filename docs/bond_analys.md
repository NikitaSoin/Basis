# Методика: оценка соответствия доходности облигации её риску

Версия 1.3. Назначение: единый алгоритм для вкладки «Доходность vs риск» в карточке облигации. Методика отвечает на один вопрос: **оплачен ли риск этой бумаги её текущей доходностью** — и даёт вердикт с аргументами, а не «простыню».

> Изменения в 1.3: добавлены РЕЖИМЫ для флоатеров (оценочная YTM от прогноза ЦБ по ставке + чувствительность эмитента к КС) и валютных бумаг (сценарная рублёвая доходность через паритет инфляций с оговорками про капитальные ограничения и реальную ставку) — раздел 4-бис; тип купона выводится в карточку.
> Изменения в 1.2: формат выдачи стал ДВУХЭТАЖНЫМ — краткий вердикт + ОБЯЗАТЕЛЬНЫЙ полный структурированный разбор по всем блокам A–F с оценками, фактами и таблицей сборки Risk Score (краткие тезисы без полного разбора = НЕ готово).
> Изменения в 1.1: требуемый спред считается по группе **рейтинг + отрасль (+ дюрация)**, а не только по рейтингу; добавлена обработка малых групп и честная оговорка про природу метода (relative value — сравнение с похожими бумагами, а не с абсолютным «справедливым» уровнем).

-----

## 0. Главная идея (логика всей методики)

Доходность облигации сама по себе ничего не говорит. Смысл имеет только **спред** — превышение доходности над безрисковой ставкой того же срока. Спред — это цена, которую рынок платит инвестору за риск. Методика состоит из трёх шагов:

1. **Посчитать, сколько рынок платит** (фактический спред бумаги).
2. **Посчитать, сколько он ДОЛЖЕН платить** за реальный риск эмитента и выпуска (требуемый спред).
3. **Сравнить** и вынести вердикт: премия (риск оплачен с запасом) / справедливо / дисконт (риск не оплачен).

Все блоки ниже работают на один из этих трёх шагов. Ничего «для красоты» в выводе быть не должно.

**Важная оговорка о природе метода (честность перед пользователем).** Требуемый спред мы калибруем по спредам похожих бумаг на рынке (relative value). Это отвечает на вопрос «дорого/дёшево относительно сопоставимых бумаг», а НЕ «дорого/дёшево относительно абсолютно справедливого уровня». Если весь сегмент (например, все ВДО) переоценён или недооценён целиком — сравнение с медианой этого не покажет, потому что медиана и есть отражение того же рынка. Поэтому вердикт всегда дополняется проверкой ожидаемыми потерями (шаг 3.3) — это независимый якорь от фундамента, а не от рынка. Где возможно, расхождение нашей оценки с рыночной и есть ценность второго мнения.

## Шаг 1. Фактическая плата за риск

### 1.1. Правильная доходность

- База — **YTM по текущей цене** (эффективная, с учётом реинвестирования купонов).
- Если есть **оферта (put)** — считать **доходность к оферте (YTP)** и использовать её как основную: до оферты денежный поток гарантирован условиями, после — купон неизвестен. Это критично для ВДО, где после оферты эмитент может поставить купон 0,01%.
- Если есть **call** — консервативно брать худшую из YTM/YTC (yield-to-worst).
- **Флоатеры** — отдельный режим: фактическая плата за риск = спред купона к базовой ставке (КС/RUONIA) с поправкой на цену. Не сравнивать YTM флоатера с YTM фикса напрямую.
- **Амортизация** — учитывать в денежном потоке (дюрация короче, чем кажется по дате погашения).

### 1.2. Фактический спред (G-spread)

```
G-spread = YTM(YTP) бумаги − доходность ОФЗ той же дюрации (точка на G-кривой)
```

- Сравнивать по **дюрации**, не по дате погашения.
- Источник кривой ОФЗ: КБД МосБиржи (G-curve). Если в БД её нет — приближение через ближайшие по дюрации ОФЗ.
- Зафиксировать в выводе: «Бумага платит X п.п. сверх безрисковой ставки».

### 1.3. Из чего состоит спред (декомпозиция для пользователя)

Спред = кредитная премия + премия за ликвидность + премия за структурные особенности (субординация, длинный срок до оферты, сложные условия). Это объясняется пользователю в одно предложение, чтобы он понимал: «40% годовых» — это не подарок, а ценник на риск.

## Шаг 2. Реальный риск: скоринг эмитента и выпуска

Риск оценивается по 6 блокам. Каждый блок даёт оценку **1–5** (1 = минимальный риск, 5 = критический) и 1–3 ключевых факта-аргумента. Блоки и веса:

|Блок                           |Вес|Что измеряет                               |
|-------------------------------|---|-------------------------------------------|
|A. Платёжеспособность (корпфин)|30%|Может ли заплатить из денег бизнеса        |
|B. Бизнес и стратегия          |20%|Устойчив ли источник этих денег            |
|C. Собственники и управление   |15%|Захотят ли платить; нет ли схем            |
|D. Макро, отрасль, PESTEL      |15%|Что среда сделает с бизнесом за срок бумаги|
|E. Параметры выпуска           |10%|Что получит держатель при проблемах        |
|F. Рынок и сентимент           |10%|Что знает рынок, чего не видно в отчётности|

### Блок A. Платёжеспособность — корпоративные финансы (вес 30%)

Это ядро. Считается по отчётности эмитента (МСФО приоритетнее РСБУ; если только РСБУ — пометить как ограничение качества данных).

**A1. Долговая нагрузка**

- Net Debt / EBITDA: <1 — низкая; 1–2 — умеренная; 2–3 — повышенная; 3–4 — высокая; >4 — критическая (для стабильных отраслей пороги мягче, для циклических — жёстче на одну ступень).
- Динамика за 2–3 года: долг растёт быстрее EBITDA — красный флаг.

**A2. Покрытие процентов**

- EBITDA / процентные расходы (ICR): >4 — комфорт; 2–4 — терпимо; 1,5–2 — напряжённо; <1,5 — компания работает на банк/держателей; <1 — проедает себя.
- При высокой ключевой ставке отдельно проверить долю долга с плавающей ставкой и график рефинансирования: дешёвый старый долг + скорое погашение = скрытый рост процентной нагрузки.

**A3. Ликвидность и график погашений (самое важное для горизонта бумаги)**

- Денежные средства + подтверждённые кредитные линии vs долг к погашению/офертам в ближайшие 12 месяцев.
- **Стена погашений**: если крупные выплаты (включая оферту по анализируемому выпуску) приходятся на один период — риск рефинансирования, даже при приличных метриках. Дефолты в ВДО почти всегда происходят на рефинансировании, а не из-за «плохого квартала».

**A4. Качество прибыли и денежный поток**

- Операционный денежный поток vs чистая прибыль: прибыль есть, а денег нет (рост дебиторки, запасов) — типичный предвестник проблем.
- FCF после капвложений: отрицательный FCF при растущем долге = долговая пирамида, пока её кто-то финансирует.

**A5. Качество баланса (анти-рисование)**

- Доля «мягких» активов в капитале: гудвил, нематериальные активы, патенты, переоценки, займы связанным сторонам. Кейс-эталон: капитал раздут до триллионов через переоценку патентов — формальные метрики прекрасны, реальных активов нет. Правило: **пересчитать ключевые метрики на ощутимый капитал** (за вычетом сомнительных НМА и займов связанным сторонам). Если картина резко ухудшается — блоку A ставится оценка не лучше 4 независимо от формальных коэффициентов.

**Оценка блока A** = худшее из (среднее по A1–A4; ограничение из A5).

### Блок B. Бизнес и стратегия (вес 20%)

Кредитор оценивает не «вырастет ли компания», а **устойчивость денежного потока на срок бумаги**.

- **Бизнес-модель в одно предложение**: на чём зарабатывает, кто платит. Если внятно не формулируется по открытым данным — это сам по себе риск-фактор.
- **Позиция на рынке**: доля, конкуренция, переговорная сила (упрощённый Портер — давление клиентов/поставщиков, лёгкость входа конкурентов).
- **Концентрация**: один клиент/госконтракт/поставщик >30% выручки — повышение оценки риска на ступень.
- **Цикличность**: строители, лизинг, ритейл-кредитование, девелопмент — в кризис первыми теряют поток; для них допустимая долговая нагрузка ниже.
- **Стратегия vs долг**: на что заняли? Расширение с понятной окупаемостью — нормально; затыкание дыр, выкуп долей, выдача займов «наверх» — красный флаг.

### Блок C. Собственники и корпоративное управление (вес 15%)

В ВДО желание платить важнее способности. Проверять:

- **Кто владеет**: конечные бенефициары известны? Цепочки офшоров/номиналов без бизнес-логики — риск.
- **Репутация**: уголовные дела, банкротства прошлых компаний бенефициара, дисквалификации, массовые суды с контрагентами (картотека арбитража — по числу и сумме исков к эмитенту).
- **Связанные стороны**: займы и сделки с аффилированными компаниями — главный канал вывода денег мимо держателей облигаций.
- **Прозрачность**: аудитор (известный/«карманный»), регулярность раскрытий, отвечает ли IR. Задержка отчётности — ранний сигнал.
- **История обслуживания долга**: тех.дефолты, переносы оферт, «реструктуризации по соглашению» в прошлом.

Любой подтверждённый факт схемного поведения (вывод активов, фиктивная отчётность) → оценка блока 5 и **общий вердикт не может быть лучше «риск не оплачен»**, какой бы ни была доходность. Это принципиально: при мошенничестве LGD стремится к 100%, и никакой спред его не компенсирует.

### Блок D. Макро, отрасль, PESTEL (вес 15%)

PESTEL применяется не «вообще», а через два канала влияния: **на денежный поток эмитента** и **на доступность рефинансирования**. На срок бумаги, не на 10 лет.

- **P (политика/гео)**: санкционная уязвимость (экспорт, импорт комплектующих, расчёты), зависимость от госзаказа, регуляторное давление на отрасль.
- **E (экономика)**: ключевая ставка — главный фактор для закредитованных эмитентов (стоимость рефинансирования) и для спроса в чувствительных отраслях (девелопмент, авто, лизинг). Курс рубля — для импортёров/экспортёров. Фаза цикла отрасли.
- **S (социум/спрос)**: устойчивость спроса на продукт, демография клиентской базы — кратко, только если материально.
- **T (технологии)**: риск устаревания модели на горизонте бумаги (для коротких бумаг почти всегда несущественно — так и писать).
- **E (экология)** и **L (право)**: экологические обязательства/штрафы для промышленности; ключевые лицензии, судебные риски, изменения регулирования (МФО, коллекторы, такси и т.п. — отрасли, где закон переписывает экономику).

Выход блока: 2–3 материальных фактора со знаком (+/−) и механизмом влияния. Факторы без механизма не включать.

### Блок E. Параметры выпуска (вес 10%)

Один эмитент — разные риски у разных выпусков.

- **Субординация / структурное подчинение**: суборды, выпуски «прокладок»-SPV без поручительства операционной компании — риск на ступень выше.
- **Обеспечение и поручительства**: кто именно поручитель и есть ли у него активы (поручительство пустышки = ничего).
- **Оферта**: ближайшая put-оферта — точка риска рефинансирования; для держателя — точка выхода. Анализировать риск к дате оферты.
- **Ковенанты**: есть ли триггеры досрочного погашения (полезно держателю) или их нет вовсе.
- **Ликвидность выпуска**: среднедневной оборот, спред bid/ask. Неликвид = невозможность выйти при первых новостях; требует надбавки к требуемому спреду (+0,5–1,5 п.п.).

### Блок F. Рынок и сентимент (вес 10%)

Рынок часто узнаёт о проблемах раньше отчётности. Сигналы:

- **Спред vs сопоставимые** (см. шаг 3.2 — группа рейтинг+отрасль+дюрация): бумага торгуется заметно шире медианы сопоставимых — рынок что-то знает или закладывает; заметно уже — спрос/неэффективность.
- **Динамика спреда**: расширение спреда конкретной бумаги при стабильном рынке — индивидуальный негатив, самый ценный ранний сигнал.
- **Поведение цены**: ступенчатые сбросы крупными объёмами — выход информированных держателей.
- **Рейтинговые действия**: не только уровень (АКРА/Эксперт РА/НКР/НРА), но прогноз и направление последнего действия; «негативный» прогноз — фактически предупреждение о понижении.
- **Новостной фон**: суды, проверки, потеря контрактов, уход аудитора, смена собственника.

## Шаг 3. Требуемый спред и вердикт

### 3.1. Интегральный риск-скор

```
Risk Score = 0.30·A + 0.20·B + 0.15·C + 0.15·D + 0.10·E + 0.10·F
```

Скор 1–5 маппится в **подразумеваемую рейтинговую группу**:

|Risk Score|Группа    |Смысл                  |
|----------|----------|-----------------------|
|1.0–1.8   |AAA–AA    |квазириск ОФЗ          |
|1.8–2.6   |A         |надёжный корп          |
|2.6–3.4   |BBB       |крепкий средний        |
|3.4–4.2   |BB–B (ВДО)|ощутимый риск дефолта  |
|4.2–5.0   |CCC и ниже|преддефолтное состояние|

Если подразумеваемая группа **хуже официального рейтинга на 2+ ступени** — в вердикте явно писать: «наш анализ видит риск выше, чем рейтинг» (и наоборот). Это и есть ценность второго мнения.

### 3.2. Требуемый спред — калибровка по сопоставимым бумагам (рейтинг + отрасль + дюрация)

Это ключевое уточнение версии 1.1. Сравнивать бумагу только по рейтинговой группе НЕ корректно: при одном рейтинге девелопер, лизинг, ритейл и промышленность несут разный риск (разная цикличность, recovery, чувствительность к ставке), и рынок закладывает им разные спреды. Поэтому требуемый спред берём по группе сопоставимых максимально близко.

**Базовая ставка требуемого спреда = медианный G-spread бумаг в группе сопоставимых из нашей базы (~3100 бумаг).** Группа определяется по трём осям, от точной к грубой:

1. **Точная группа: тот же рейтинг + та же отрасль + близкая дюрация.** Это эталон сравнения.
2. Дюрацию считать «близкой» в разумном коридоре (например ±1–1,5 года); если бумаг мало — расширить коридор.

**Обработка малых групп (обязательна — иначе медиана по 2–3 бумагам недостоверна).**
Определи минимальный размер группы для надёжной медианы (ориентир — не меньше ~5–7 бумаг; если меньше — медиана хрупкая). Алгоритм деградации, по убыванию точности:
- (a) рейтинг + отрасль + близкая дюрация — если бумаг достаточно, используй это;
- (b) если мало → рейтинг + отрасль (любая дюрация в пределах сегмента) — пометь «дюрация не учтена, точность ниже»;
- (c) если всё ещё мало → рейтинг + близкая отрасль-аналог (сгруппируй родственные отрасли: например девелопмент+стройка, лизинг+факторинг, разные виды ритейла) — пометь «отрасль укрупнена»;
- (d) если и так мало → только рейтинговая группа (как в версии 1.0) — пометь ЯВНО «сравнение только по рейтингу, без отрасли — оценка ориентировочная, точность ограничена».
В выводе ВСЕГДА указывай, на какой группе считался требуемый спред и сколько в ней бумаг (например: «медиана 18 бумаг группы BB / девелопмент / 1–2 года»). Это и есть честность about точности.

Пересчитывать медианы при обновлении котировок — это рыночная цена риска «здесь и сейчас», лучше любых вечных констант.

**Группа берётся по ХУДШЕЙ из (официальный рейтинг / подразумеваемый рейтинг Basis из 3.1)** — консервативно.

Корректировки к медиане группы (надбавки/скидки к требуемому спреду):
- неликвидность выпуска: +0,5…+1,5 п.п.
- субординация/SPV без реального поручителя: +1…+3 п.п.
- стена погашений в ближайшие 12 мес.: +1…+2 п.п.
- негативный прогноз рейтинга / расширяющийся спред: +0,5…+1,5 п.п.
- сильная ликвидная подушка, короткая дюрация до оферты: −0,5…−1 п.п.

### 3.3. Проверка здравым смыслом: ожидаемые потери (независимый якорь от фундамента)

Это второй, НЕзависимый от рынка способ — он не сравнивает с другими бумагами, а считает минимум «снизу», от вероятности дефолта. Показывать пользователю всегда (особенно ценен, когда сравнение по группе ненадёжно из-за малой выборки):

```
Минимально необходимый спред ≈ PD_годовая × LGD
```

- PD — историческая годовая частота дефолтов по рейтинговой группе (матрицы Эксперт РА/АКРА; ориентиры: BBB ~1%, BB ~2–4%, B ~5–10%, CCC >20%; уточнить по актуальным публикациям агентств — требуется верификация).
- LGD для необеспеченных ВДО консервативно 70% (возврат в российских дефолтах ВДО исторически низкий).

Пример формулировки для пользователя: «При риске дефолта ~8% в год и потере ~70% вложенного при дефолте вы должны получать минимум ~5,6 п.п. сверх ОФЗ только за ожидаемые потери — плюс премию за нервы и неликвидность. Бумага платит 12 п.п. — премия есть».

ВАЖНО — два якоря вместе: если сравнение по группе и проверка ожидаемыми потерями ПРОТИВОРЕЧАТ друг другу (по группе «справедливо», а по ожидаемым потерям спред не покрывает даже PD×LGD, или наоборот) — это сигнал, не игнорируй: укажи расхождение и склоняйся к более консервативному выводу. Совпадение обоих якорей усиливает уверенность в вердикте.

### 3.4. Вердикт

```
Премия = Фактический G-spread − Требуемый спред
```

|Премия   |Вердикт                                                                                 |Светофор|
|---------|----------------------------------------------------------------------------------------|--------|
|> +2 п.п.|Риск оплачен с запасом (возможна неэффективность — проверить, не знает ли рынок чего-то)|🟢       |
|+0.5…+2  |Риск оплачен                                                                            |🟢       |
|−0.5…+0.5|Справедливо                                                                             |🟡       |
|−2…−0.5  |Риск недоплачен                                                                         |🟠       |
|< −2 п.п.|Риск существенно недоплачен                                                             |🔴       |

**Стоп-правила (перекрывают арифметику):**

1. Подтверждённые схемы/мошенничество (блок C=5) → вердикт не лучше 🔴, с прямым объяснением: «доходность не компенсирует риск потерять всё».
2. ICR < 1 и стена погашений ≤ 6 мес. без понятного источника рефинансирования → не лучше 🟠.
3. Большая положительная премия (>4 п.п. к группе) — НЕ «подарок», а повод спросить «что знает рынок?»: вердикт 🟢 только если блоки C и F чистые. Если премия аномально велика (бумага платит кратно больше группы) — это почти всегда сигнал высокого риска дефолта/неликвида, который рынок видит, а не «выгодная сделка»; по умолчанию трактуй как 🟠/🔴 с пометкой «рынок оценивает риск намного выше — разберись почему», а не как 🟢. Зелёный «с запасом» допустим ТОЛЬКО когда качественные блоки (A–F) подтверждают, что риск действительно умеренный, а широкий спред объясняется неэффективностью/неликвидом, а не близостью к дефолту.
4. Дефолт/техдефолт в истории или действующий → доходность нерелевантна как «премия», вопрос смещается к recovery; вердикт не лучше 🟠 с явным указанием на состоявшийся дефолт.
5. Аномальный YTM (>40% годовых) → маркер дистресса/неликвида, не «высокая доходность»: проверь near-offer артефакт (доходность к близкой оферте раздувается технически — НЕ путать с дистрессом) и неликвид (бумажная доходность от старой цены); по умолчанию НЕ 🟢.

## 4. Формат выдачи — ДВА ЭТАЖА: вердикт + полный разбор

Выдача обязана содержать ОБА этажа. Краткий вердикт без полного разбора под ним —
НЕ готово: тезисы без видимого фундамента не вызывают доверия. Полный разбор без
вердикта сверху — простыня. Нужны оба.

### Этаж 1 — Вердикт (ответ за 10 секунд, всегда виден сверху)

1. **Вердикт-плитка**: светофор + одна фраза («Доходность 27% платит за риск с запасом ~2 п.п.» / «38% не покрывают преддефолтный риск»).
2. **Арифметика в 3 строки**: фактический спред (из системного расчёта, не прикидка) → требуемый спред (с указанием группы сравнения и ЧИСЛА бумаг в ней; при деградации группы — на каком уровне) → премия/дисконт.
3. **3–4 главных аргумента**, каждый строго *факт → механизм → вывод* — самые материальные из блоков A–F (это выжимка из Этажа 2, не замена ему).
4. **Проверка ожидаемыми потерями** (строка с расчётом) — и пометка, если расходится со сравнением по группе.
5. **Триггеры пересмотра**: 1–2 наблюдаемых события.

### Этаж 2 — Полный разбор по блокам (разворачиваемый, ОБЯЗАТЕЛЕН)

По КАЖДОМУ блоку A–F — отдельная структурированная секция:
- **Заголовок блока + его оценка 1–5** (видна сразу в заголовке секции).
- **Детальные факты** списком, каждый: факт с числом (источник, дата) → механизм →
  что это значит для оценки блока. Для блока A — пройтись по подпунктам A1–A5
  (долговая нагрузка с динамикой, ICR, ликвидность и стена погашений, качество
  потока, анти-рисование); для B — модель/позиция/концентрация/цикличность/на что
  заняли; для C — владельцы/репутация/связанные стороны/прозрачность/история
  долга; для D — 2–3 материальных фактора с механизмом; для E — оферта/
  субординация/поручители/ковенанты/ликвидность выпуска; для F — спред vs группа/
  динамика/рейтинговые действия/новости.
- Где данных по подпункту нет — честно «нет данных» (это тоже информация), не
  пропускать молча.

### Сборка оценки (мост от разбора к вердикту, ОБЯЗАТЕЛЕН)

Таблица: блок → оценка 1–5 → вес → вклад. Итог: **Risk Score → подразумеваемая
группа → требуемый спред → вердикт**. Пользователь должен видеть, как вывод ЧЕСТНО
сложился из разобранных блоков через веса, а не возник из воздуха. Если применено
стоп-правило — явно показать, какое и почему оно перекрыло арифметику.

### Запрещено
Неструктурированная простыня (текст без секций и заголовков); оценки без фактов;
факты без механизма; выводы «с одной стороны — с другой» без итога; КРАТКИЙ
вердикт БЕЗ полного разбора под ним (главная ошибка: глубина — это и есть
ценность, «анти-простыня» означает «структурированно», а НЕ «поверхностно»).

## 4-бис. Режимы для НЕ-фикс облигаций (флоатеры и валютные) — v1.3

Базовая методика считает спред к рублёвой ОФЗ-кривой и предполагает фиксированный
рублёвый купон. Для флоатеров и валютных бумаг прямой G-spread к ОФЗ
НЕприменим/искажён. Но НЕ помечай их просто «неприменимо» — оцени по
адаптированному режиму. Тип купона определяй по BOND_TYPE MOEX и условиям выпуска.

### Режим A. ФЛОАТЕРЫ (купон привязан к КС/RUONIA)

Ключевая мысль: у флоатера процентный риск держателя минимален (купон плавает с
рынком), поэтому ГЛАВНЫЙ риск — кредитный (эмитент), плюс ДВА дополнительных
канала: (1) рост КС удорожает долг самого эмитента; (2) будущий купон неизвестен,
зависит от траектории КС.

**Доходность — оценочная YTM от прогноза ЦБ:**
1. Возьми среднесрочный прогноз ЦБ по ключевой ставке (диапазон на 1–2 года из
   ОНЕГДКП / пресс-релиза последнего заседания; средняя КС по годам).
2. Подставь траекторию КС в формулу купона (например КС + X%) → ожидаемые купоны
   по периодам.
3. Посчитай YTM от этих ожидаемых купонов и текущей цены → это ОЦЕНОЧНАЯ YTM.
   Пометь явно «оценочная YTM, от прогноза ЦБ по ставке» (это не факт, а
   judgement — будущие купоны зависят от ставки).
4. Прогноз ЦБ — ДИАПАЗОН → дай оценочную YTM диапазоном или от середины с
   пометкой о чувствительности к траектории КС.
5. От оценочной YTM считай спред к ОФЗ сопоставимой дюрации (как в базовой
   методике) → требуемый спред → вердикт. Но помни: спред флоатера в первую
   очередь про КРЕДИТНЫЙ риск, процентный почти снят.

**Риск (блок A с поправкой):** проверь чувствительность ЭМИТЕНТА к росту КС —
долю долга с плавающей ставкой, ICR при сценарии КС +2 п.п. (если ставка вырастет,
покроет ли проценты). Для эмитента-флоатера это прямой канал ухудшения.

**Подача:** в карточке — формула купона явно (КС + X%), оценочная YTM с пометкой,
вердикт от кредитного риска. Не выдавай оценочную YTM за факт.

### Режим B. ВАЛЮТНЫЕ (замещающие / в USD/CNY)

Ключевая мысль: доходность валютной бумаги для рублёвого инвестора =
валютная YTM + ПЕРЕОЦЕНКА КУРСА. Сравнивать её валютную YTM с рублёвой ОФЗ
напрямую НЕЛЬЗЯ — нужен прогноз курса. Это самая сужденческая часть методики —
подача ОБЯЗАТЕЛЬНО СЦЕНАРНАЯ, без ложной точности.

**Оценка ожидаемой рублёвой доходности:**
1. Валютная YTM бумаги (в USD/CNY) — факт из рынка.
2. Прогноз изменения курса через паритет инфляций (ориентир, не точность):
   при прочих равных рубль слабеет примерно на разницу инфляций (ИПЦ РФ − ИПЦ
   США/Китая). ОГОВОРКИ (обязательно проговорить): канал работает не идеально
   (капитальные ограничения, структурные факторы); высокая РЕАЛЬНАЯ ставка
   (КС − инфляция РФ) в моменте работает в сторону укрепления рубля — то есть
   краткосрочно курс может идти против паритета.
3. ДВА сценария (минимум):
   - «Стабильный курс»: рублёвая доходность ≈ валютная YTM (переоценки нет).
   - «Ослабление на разницу инфляций»: рублёвая доходность ≈ валютная YTM +
     ожидаемое ослабление (например +3–4%/год).
4. Сравни рублёвую доходность валютной бумаги (по сценариям) с доходностью
   сопоставимой по риску РУБЛЁВОЙ бумаги. Вывод: «валютная доплачивает за
   валютный риск» / «при ослаблении рубля выигрывает, при стабильном — рублёвая
   альтернатива выгоднее» и т.п. — сценарно.

**Риск (блок A с поправкой):** валютная структура долга vs выручки ЭМИТЕНТА.
Валютный долг при рублёвой выручке — риск (рост курса удорожает обслуживание);
валютная выручка под валютный долг — естественный хедж.

**Подача:** валюта номинала явно; валютная YTM (факт) + сценарии рублёвой
доходности (judgement, сценарно) — НЕ одна цифра. Прогноз курса всегда помечен
как наиболее сужденческая часть.

### Near-maturity / near-offer
Короткий хвост до погашения/оферты механически раздувает YTM — НЕ риск. Если
до погашения/ближайшей оферты осталось мало (артефакт near-maturity/near-offer):
краткая пометка «доходность искажена близостью погашения/оферты», без вердикта
по спреду. (Уже зашито в стоп-правило 5 и в bond_risk.py.)



- Полные данные (МСФО + рейтинг + ликвидные котировки) → полная методика.
- Только РСБУ → блок A считается с пометкой «по РСБУ, качество ограничено», A5 (анти-рисование) обязателен.
- Нет рейтинга → подразумеваемая группа только по нашему скору; требуемый спред — по группе скоринга; пометка «без официального рейтинга — риск-премия повышена» (+1 п.п. к требуемому спреду).
- Нет свежей отчётности (>12 мес.) → это сам по себе красный флаг: блок C не лучше 4, явная пометка пользователю.
- **Малая группа сравнения** (см. 3.2): всегда указывай размер группы и уровень деградации (рейтинг+отрасль+дюрация → … → только рейтинг); при только-рейтинге опирайся сильнее на проверку ожидаемыми потерями (3.3) как на независимый якорь.
- **Неликвид** (нет цены сделки / нет спреда / 3-й уровень без оборота): полный разбор «доходность vs риск» не строится осмысленно — доходность «бумажная». Достаточно краткой пометки «бумага неторгуемая/редкоторгуемая, рыночная доходность недостоверна, оценить соотношение доходность/риск нельзя», без светофора-вердикта.
- Каждый вывод обязан различать: факт (из отчётности/котировок), оценку рынка (спреды/рейтинги) и нашу гипотезу.
