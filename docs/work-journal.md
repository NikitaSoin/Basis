### 2026-06-11 — РЕДИЗАЙН v1.2: двухэтажный формат + структурный рендер (запрос владельца)
Методичка обновлена до v1.2 (§4): формат ДВУХЭТАЖНЫЙ. Этаж 1 = вердикт (светофор,
арифметика с группой+N бумаг, 3–4 аргумента, потери, триггеры). Этаж 2 ОБЯЗАТЕЛЕН =
полный разбор по КАЖДОМУ блоку A–F (оценка 1–5 в заголовке + факты A1–A5 и т.д. →
механизм → значение) + таблица СБОРКИ (блок→оценка→вес→вклад→Risk Score→группа→вердикт
+ стоп-правило). Краткие тезисы без Этажа 2 = НЕ готово. ВТОРОЕ: рендер «сырым
markdown одной плитой» → структурный по дизайн-системе (как вкладки Финансы/Геополитика).

АРХИТЕКТУРНОЕ РЕШЕНИЕ (моё): данные = КАНОН-СТРУКТУРНЫЙ MARKDOWN (не JSON) — сохраняет
«текст в файлах», человекочитаемо/правимо, а ЖЁСТКАЯ схема заголовков+токенов делает
парсинг детерминированным на 388 файлов. Канон-схема: секции `## ВЕРДИКТ {light:X}` /
`## АРИФМЕТИКА` / `## ГЛАВНЫЕ АРГУМЕНТЫ` / `## ПРОВЕРКА ПОТЕРЯМИ` / `## ТРИГГЕРЫ` /
`## БЛОК A…F — … {score:N}` / `## СБОРКА` (md-таблица) / `## СНОСКИ` (с `{warn}`) /
`## ИСТОЧНИКИ`. Inline-метки `{факт}/{оценка}/{суждение}`, `{warn}`. light ∈
{red,orange,amber,green,gray}; score ∈ 1..5. Фронт парсит по `## ` и токенам.
