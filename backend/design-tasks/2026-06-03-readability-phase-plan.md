# План фазы «Читаемость плотного контента» (только план)

Проблема: текстовые блоки (Обзор — комментарий аналитика, Корп. управление,
Бизнес-модель) поданы простынёй мелкого текста. Перевод на примитивы причесал
рамки, но НЕ улучшил читаемость. Цель фазы — сделать плотный текст НАВИГИРУЕМЫМ.

## ⛔ ГЛАВНОЕ ПРАВИЛО (не нарушать)
**Содержание НЕ выхолащивать.** Оговорки, нюансы, «честно: данные противоречивы»,
объяснения почему метод ненадёжен — это ЦЕННОСТЬ продукта (второе мнение,
честность про неопределённость), НЕ вода. **Не сокращать смысл, не выкидывать
нюансы, не урезать абзацы.** Задача — сделать длинный текст легко СКАНИРУЕМЫМ:
видно суть сразу, детали — рядом, за один клик/взгляд. Меньше текста = провал.

## Приёмы (применять единообразно ко всем текстовым блокам)
1. **Вывод вперёд (BLUF — bottom line up front).** В начале каждого блока/секции —
   1-2 фразы главного вывода КРУПНО/выделенно (lead). Полное рассуждение — ниже,
   мельче. Пользователь читает вывод, при желании углубляется.
2. **Числа из текста → визуалы.** Любое число/доля/диапазон, утопленные в прозе,
   выносим в KpiTile/полосу/мини-таблицу/дельту (эталоны уже в продукте: Мини-P&L
   и таблица методов оценки — ровно та подача). Текст остаётся, но число продублировано
   наглядно и «цепляет глаз».
3. **Длинные рассуждения → сворачиваемые секции** с понятным заголовком-саммари:
   заголовок = одна фраза сути (видно, о чём), развернул — полный текст со всеми
   оговорками. Ничего не теряется, но экран не забит.
   **⚠️ Сворачивать ТОЛЬКО детали второго порядка** — то, без чего смысл блока не
   теряется. **Ключевые выводы (LeadStatement) и «честные» оговорки про
   неопределённость (KeyTakeaway) держим РАСКРЫТЫМИ по умолчанию** — это ценность
   продукта, она должна быть НА ВИДУ, а не за кликом. Под `Disclosure` уходят:
   расширенные обоснования, второстепенные факторы, методологические подробности,
   длинные перечни — но НЕ сам вывод и НЕ оговорка про неопределённость.
4. **Типографика-воздух.** Межстрочный ~1.6, ограничение ширины строки (~70 символов
   / `max-w-prose`), разбивка на абзацы по смыслу, отступы между абзацами по 8pt.
   Сейчас мелко и сплошняком — поднять размер body до 14-15px, дать воздух.
5. **Выделение ключевых фраз-выводов.** Внутри абзаца ключевую фразу-вывод —
   полужирным/акцентным фоном (callout). Особо — «честные» оговорки про
   неопределённость: дать им заметный, но не тревожный вид (info/нейтраль-callout
   «⚖ Honest take»), чтобы это читалось как фича доверия, а не сноска мелочью.

## Переиспользуемые компоненты (новые примитивы текста — собрать в src/design/)
- `<Prose>` — обёртка для длинного текста: межстрочное, max-w, ритм абзацев,
  стили списков/выделений (markdown тоже через него).
- `<LeadStatement>` — крупный вывод-лид сверху блока (h3-h4, акцент).
- `<KeyTakeaway tone="info|positive|caution">` — callout для ключевой фразы/оговорки
  («Честно: данные противоречивы…»). Тон info/нейтраль для «честности про
  неопределённость» (НЕ warning-тревога).
- `<Disclosure summary="...">` — сворачиваемая секция с заголовком-саммари (нативный
  details/summary, доступно, reduced-motion ок). Развёрнутый текст — полный, 1:1.
- `<StatInline value unit label>` — извлечённое из текста число как мини-визуал
  (рядом с прозой или вместо повтора в тексте).

## ОБРАЗЕЦ 1 — Обзор, «Комментарий аналитика / Что происходит»
СЕЙЧАС: абзац-простыня мелким текстом (что происходит с акцией, факторы, оговорки).

СТАЛО (схема):
```
┌─ Что происходит с акцией ───────────────────────────────┐
│ ▎ ЛИД (крупно, акцент):                                  │
│ ▎ «Акция под давлением из-за роста ставки ЦБ, но         │
│ ▎  фундаментально недооценена по историческому P/E.»     │
│                                                          │
│  Полный комментарий (Prose, воздух, max-w):              │
│  Сбербанк остаётся ключевым… [абзац 1, все нюансы] …     │
│  [абзац 2: процентная маржа, качество активов] …         │
│                                                          │
│  ⚖ Честно (KeyTakeaway, info-callout):                   │
│  «Данные по стоимости риска противоречивы: при жёсткой   │
│   ДКП маржа растёт, но кредитный рост тормозит — итог     │
│   зависит от траектории ставки, которую точно не         │
│   предсказать.»                ← оговорка ВЫДЕЛЕНА, не    │
│                                   спрятана в простыне     │
│                                                          │
│  ▸ Подробнее: чувствительность к ставке (Disclosure)     │  ← свёрнуто
└──────────────────────────────────────────────────────────┘
```
Принцип: вывод сразу; полный текст и ВСЕ оговорки на месте; «честная» оговорка
поднята в заметный callout (это фича доверия); глубокая детализация — под раскрытие.

## ОБРАЗЕЦ 2 — Бизнес-модель, секция (напр. «Структура выручки» / «Конкурентная позиция»)
СЕЙЧАС: заголовок + абзацы текста; доли уже частично визуализированы полосами (хорошо).

СТАЛО (схема):
```
┌─ Конкурентная позиция ──────────────────────────────────┐
│ ▎ ЛИД: «Широкий ров за счёт экосистемы и доли рынка      │
│ ▎  ~32%, но регуляторное давление — главный риск.»       │
│                                                          │
│  Преимущества            Уязвимости                      │
│  ✓ Экосистема (success)  ✗ Регулирование (danger)        │  ← уже визуалы, усилить
│  ✓ Доля рынка 32% ▕▕▕▕▕░ ✗ Завис. от ставки ▕▕▕░░        │
│                                                          │
│  Полный разбор (Prose): … [все абзацы, нюансы] …         │
│                                                          │
│  ▸ Драйверы маржи — детали (Disclosure)                  │  ← длинное рассуждение свёрнуто
└──────────────────────────────────────────────────────────┘
```
Числа (доля рынка, метрики) — продублированы полосами/StatInline; длинные
рассуждения о драйверах — под раскрытие с понятным саммари-заголовком.

## Раскатка (порядок, ПОСЛЕ утверждения подхода)
1. Собрать текстовые примитивы (`Prose/LeadStatement/KeyTakeaway/Disclosure/StatInline`)
   на `/_design` (витрина, как делали для остальных) — утвердить вид.
2. Применить к **Обзору** (комментарий аналитика + макро/политика-блоки) — push → reviewer → проверка.
3. Применить к **Бизнес-модели** (markdown-секции + структурный профиль) — push → reviewer → проверка.
4. Применить к **Корп. управлению** (длинные notes: transparency, related_party,
   policy_vs_practice, прецеденты) — push → reviewer → проверка.
5. (Опц.) Финансы/оценка — там в основном уже визуалы; подчистить текстовые сноски.

## Definition of Done фазы
- Ни один блок не потерял смысла/оговорок/нюансов (дифф текста = 0 удалённых
  предложений; только переоформление/выделение/сворачивание).
- В каждом блоке: вывод сверху, числа продублированы визуалом, длинное — под
  раскрытие с осмысленным заголовком, воздух/межстрочное добавлены, «честные»
  оговорки выделены как фича.
- Обе темы, доступность (details/summary с клавиатуры, reduced-motion), AA.

> Важно: это фаза ПОДАЧИ, не редактуры. Тексты не переписываем и не сокращаем —
> только структурируем и выделяем. Если где-то текст реально слабый — это вопрос
> к контент-агентам (business-model-analyst и т.п.), не к этой фазе.
