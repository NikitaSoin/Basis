# Аудит карточки KRKNP (ПАО «Саратовский НПЗ», привилегированные) — card-critic

## 1. ВЕРДИКТ ЦЕЛОГО

**4/10 — качественные вкладки добротные, но финансовый блок РАЗДВОЕН: `financials.json` и `financials_summary.md` — это две РАЗНЫЕ, противоречащие друг другу оценки одной бумаги. JSON не пересобран под преф и остался копией обыкновенной KRKN.**

KRKNP — тот же эмитент, что KRKN, и качественные вкладки (бизнес-модель, рынки, макро, гео, управление) корректно переиспользованы с правильным префовым акцентом (дивиденд = 10% РСБУ-прибыли, без голоса). Но ядро карточки преф — оценка дивиденда — сломано на уровне данных: витрина (`financials.json`, который рендерит сайт) показывает оценку ОБЫКНОВЕННОЙ акции, а текстовое резюме — совсем другую дивидендную оценку. Инвестор увидит два разных fair value на одну бумагу.

## 2. КАРТОЧКА КАК ЦЕЛОЕ

### КРИТИЧНО: JSON ≠ summary — две несовместимые оценки
`financials.json` KRKNP — побайтово почти копия KRKN:
- `shares_outstanding = 997349` (оба класса, а не 249 337 префов);
- `fair_value_range = {cons 5858, base 8357, opt 10729}` — это оценка ОБЫКНОВЕННОЙ KRKN через P/E/P/B/CAPM;
- `dividend_yield = not_applicable` — для ПРЕФА, единственной дивидендной бумаги эмитента, дивидендный метод помечен «неприменим». Это абсурд: для префа он должен быть ОСНОВНЫМ.

`financials_summary.md` рассказывает совершенно другое: цена префа 6 780 ₽, дивидендный метод как ядро, fair base 2 984 ₽ / cons 2 238 ₽ / низ 1 584 ₽, даунсайд −56…−77%, DPS 447,6 ₽, P/E 6.1, дивдоходность 6.6%.

Эти два артефакта одной карточки несовместимы: один даёт fair 5858–10729 (апсайд/около цены), другой 1584–2984 (даунсайд −56…−77%). Это не нюанс — это фундаментальное противоречие внутри одной вкладки. JSON явно не перегенерирован под преф; summary написан отдельно и вручную. На сайт пойдёт JSON — то есть инвестор увидит оценку НЕ ТОЙ бумаги.

### Третий слой противоречия — business_model.md
`business_model.md` KRKNP оценивает «~11 млн привилегированных акций» и «~10 руб./акцию» дивиденда. Но и governance, и summary, и сам устав дают 249 337 префов и 447,54 ₽/преф. Расхождение в число акций в ~44 раза. В одной карточке гуляет ТРИ числа префов: 249 337 (верно), 997 349 (JSON, оба класса), ~11 млн (бизнес-модель). И три DPS: 447,54 / ~10 / 344.

### Связность качественных вкладок — нормально
Бизнес-модель, рынки, макро, гео, управление согласованы между собой и с KRKN: процессинг, тариф Роснефти, кубышка, удары БПЛА, асимметрия классов. Префовый акцент проведён (дивиденд — центр истории). Если бы не разъехавшийся финансовый блок, карточка была бы на уровне KRKN.

### Целостность портрета
Из качественных вкладок инвестор поймёт суть. Но как только дойдёт до «сколько стоит и какой дивиденд» — получит взаимоисключающие числа. Для дивидендной бумаги это смертельно: весь смысл префа — дивиденд, и именно тут карточка себе противоречит.

## 3. ПО ВКЛАДКАМ

**Финансы — 2/10.** Summary как текст хорош: дивидендный метод правильно выбран ядром, нормализация на медиане ЧП, честный вывод «цена в 2–3 раза выше фундаментала, оценка структурно ненадёжна». НО он не бьётся с `financials.json`, который рендерится на сайте и содержит оценку обыкновенной акции с dividend_yield=not_applicable. Пока JSON не пересобран под преф, вкладка выдаёт неверные данные. Дополнительно: ряд ЧП в summary (2820→980→600→600→1116) расходится с KRKN/JSON (там 2020–2022 = null/оценка); FCF по МСФО (180/120 млн) появился только в summary, в JSON cash_flow весь null.

**Бизнес-модель — 6/10.** Правильный префовый разворот, но грубая ошибка числа префов (~11 млн вместо 249 337) и вытекающего DPS (~10 ₽ вместо 447,54 ₽). Тест «стоит ли брать ради дивиденда» полезен, но опирается на неверную базу акций.

**Рынки — 7/10.** Переиспользовано с KRKN, корректно (доля мощностей, а не рынок топлива; остановка АВТ-6).

**Макро — 7/10.** Тот же верный разворот (ставка в плюс через кубышку, риск — инфляция издержек vs тариф). Связан с дивидендом префа явно.

**Гео — 7/10.** Сценарная развилка по ударам БПЛА, SDN на Роснефть. Корректно увязано с риском дивиденда префа.

**Управление — 7/10.** weak-флаг, асимметрия классов, преф предпочтительнее обыкновенной по денежному потоку, но база управляема группой. Хорошо.

## 4. ФАКТИЧЕСКИЕ ОШИБКИ / ПРОТИВОРЕЧИЯ

1. **`financials.json` KRKNP = копия KRKN (обыкновенной)**: fair 5858–10729, dividend_yield=not_applicable, shares 997349. Для префа это неверная оценка. (Финансы — критично, идёт на сайт)
2. **JSON (fair 5858–10729) vs summary (fair 1584–2984)** — взаимоисключающие fair value одной бумаги. (Финансы)
3. **Число префов: 249 337 (верно) vs 997 349 (JSON) vs ~11 млн (бизнес-модель)**. (Финансы vs Бизнес-модель)
4. **DPS: 447,54 ₽ (factual) vs ~10 ₽ (бизнес-модель) vs 344 ₽ (приходит из KRKN-карты)**. (Бизнес-модель vs Финансы)
5. **Ряд ЧП 2020–2022: 2820/980/600 (summary) vs null/оценка (JSON)** — источник не сверен. (Финансы)
6. **FCF по МСФО 180/120 млн в summary, но в JSON cash_flow весь null** — данные есть в тексте, нет в витрине. (Финансы)
7. **last_price=null в JSON, но summary оперирует ценой 6 780 ₽** — цена не зафиксирована в данных. (Финансы)

## 5. ЧЕГО НЕ ХВАТАЕТ ДО ЭКСПЕРТНОСТИ

1. **ПЕРЕСОБРАТЬ `financials.json` под преф**: shares=249 337; dividend_yield — ОСНОВНОЙ метод (status ok) с fair 1584–2984; P/E/P/B/CAPM по обыкновенной — убрать или пометить not_applicable. Сейчас витрина показывает чужую оценку.
2. **Свести JSON и summary к одному набору чисел** (цена, DPS, fair, даунсайд).
3. **Исправить число префов в бизнес-модели** (249 337, DPS 447,54 ₽).
4. **Зафиксировать last_price** в данных, чтобы выводы о даунсайде были воспроизводимы.
5. **Сверить ряд ЧП и FCF** между summary и JSON по одному источнику.

---
*Системная проблема: финансовый блок КRKNP существует в двух несовместимых версиях — текстовое резюме описывает правильную дивидендную оценку префа, а машинная витрина (`financials.json`) осталась копией оценки обыкновенной акции; на сайт уходит неверная.*

**Оценка: 4/10. Системная проблема: `financials.json` префа не пересобран и противоречит собственному резюме — витрина показывает оценку обыкновенной KRKN (fair 5858–10729, дивметод «неприменим») вместо дивидендной оценки префа (fair 1584–2984).**
