Вкладка «Институты» карточки компании переведена из classic в NEO отдельным
компонентом `frontend/Basis/src/company/InstitutionsTab.jsx` +
`frontend/Basis/src/styles/institutions.css` (по образцу GovernanceTab/FinanceTab/
BusinessModelTab). App.js: `renderInstitutions` теперь только guard (instLoading /
`!instMd && !instJson` → renderComingSoon) + `return <InstitutionsTab .../>`.

**Главная находка (важно при любой будущей работе с institutions.json/agent):**
15 субиндексов IRI (`iri_scoring.subindices`, ключи S1..S15) — ключи НЕ означают
одно и то же у разных компаний. Системный промпт `institutional-company-analyst.md`
не фиксирует canonical-список (только «S7 = Coalition Position» явно закреплён в
базе знаний `docs/Институты_агенты.md`) — каждый прогон агента сам присваивает
порядок/метки по шкалам методики. Проверено программно на ВСЕХ 16 обработанных
компаниях (`backend/companies/*/institutions.json`): S7 действительно всегда про
клановую позицию, но S1/S8/S14/etc. — совершенно разные концепты у разных
компаний (напр. SBER.S1 = «защита прав собственности», GAZP.S1 = «прозрачность
структуры собственности», TATN разносит клановую тему на S5+S6+S7 сразу три
субиндекса). Позиционная группировка «S1,S2,S3,S13 → кластер X» (буквально то,
что просил ТЗ design-critic) даёт корректный результат ТОЛЬКО для SBER (видимо
эталон, на который смотрел critic) и ломается почти everywhere ещё.

**Решение:** кластеризация в UI — по regex-классификатору смысла `label`
(`classifyDim()` в InstitutionsTab.jsx, ~8 приоритетных правил, порядок важен:
нерезидент→E раньше общего «ликвидность»→G, фискальные ключевые слова раньше
generic «изъят»→P и т.д.). Даёт 6 пакетов: P (права собственности/суд),
E (внешний контур/санкции), CLAN (всегда=S7-подобное), F (фискально-политическая
нагрузка), G (governance/«сверка», = бывший S8-S11), T (траектория). Верифицировано
скриптом на всех 240 реальных субиндексах (16×15) — 0 unmatched, для SBER состав
пакетов побитово совпал с буквальной S-key таблицей critic'а (подтверждает, что
классификатор реализует ЗАМЫСЕЛ, а не отступает от него). `Cluster`-компонент
решает alwaysOpen ДИНАМИЧЕСКИ (`items.length <= 1`), а не хардкодом «клан всегда 1
пункт» — важно, т.к. TATN даёт 3 пункта в CLAN-пакете.

**Если будущая задача трогает institutions.json схему или сам агент:** стоит
рассмотреть закрепление canonical S1-S15 в системном промпте
`institutional-company-analyst.md` (сейчас только S7 зафиксирован) — тогда
классификатор в UI можно будет упростить обратно к позиционному. Пока не сделано
— это НЕ входило в задачу дизайн-редизайна, отмечено как находка для владельца.

Другие интерпретационные решения (расходятся с буквальным ТЗ design-critic,
задокументированы в чате/коммите): вердикт-герой обрезается word-safe (`shortText`,
не sentence-regex) из-за пилюли «2,0 п.п.» — точка внутри аббревиатуры ломает
naive-сентенс-детект; «Сверка с governance» — заголовок без «(S8-S11)» в тексте
(число факторов там теперь динамическое); `onNavigateTab` поддержан компонентом,
но не прокинут из App.js (литеральный вызов в ТЗ был без него).

См. также [[bond-risk-render]] (аналогичный паттерн canonical-парсинг+легаси-фолбэк
для другого сложного аналитического блока).
