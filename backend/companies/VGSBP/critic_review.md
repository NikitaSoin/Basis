# Аудит карточки VGSBP (Волгоградэнергосбыт, прив.) — card-critic

## 1. ВЕРДИКТ ЦЕЛОГО
**3/10 — преф разобран как копия обычки: финансовая модель не переведена на префовую логику (дивиденд отключён, fair value = числа обычки), а блоки расходятся между собой по цене, методу и выводам.**

Качественные блоки (бизнес-модель, макро, гео, управление) для префа написаны грамотно и даже с правильным акцентом «префы — это про дивиденд, обычка — нет». Но финансовый блок — ядро карточки оценки — фактически не сделан для префа: financials.json дословно повторяет VGSB, а ключевой для привилегированной акции метод (дивидендный) отключён с формулировкой про обычку. Карточка как продукт внутренне противоречива и вводит инвестора в заблуждение по справедливой цене.

---

## 2. КАРТОЧКА КАК ЦЕЛОЕ

### Критично — финблок префа = механическая копия обычки
`financials.json` VGSBP идентичен VGSB во всём, что касается оценки:
- `valuation.fair_value_range` = **16,6 / 19,3 / 27,5** — это значения для **обычки** (цена обычки 16,89 ₽).
- `historical_pe.fair_value_per_share` = **19,32**, CAPM = **30,72**, шаги расчёта оперируют ценой **16,89 ₽** и «капитализацией 5397 млн».
- `multiples.current.pe` = 2,52 — на цене обычки.

При этом «шапка» summary заявляет: «мультипликаторы и справедливая стоимость **пересчитаны на коэффициент цены префа к обычке 0.96**, цена префа 13,15 ₽». Этот пересчёт в JSON **не выполнен** — все числа остались обычкины. Инвестор, открывший вкладку «Финансы» префа, увидит fair value 16,6–27,5 ₽ против «цены префа 13,15 ₽» — апсайд завышен искусственно, потому что fair value считался от цены другой бумаги. Это прямая фактическая ошибка, подрывающая весь смысл отдельной карточки префа.

### Критично — дивидендный метод отключён неверным обоснованием
Для **привилегированной** акции дивиденд — это главный (часто единственный осмысленный) метод оценки. В JSON метод `dividend` стоит `not_applicable` с caveat: «по **обыкновенным** акциям VGSB дивиденды не выплачиваются с ~2011 г.». Это рассуждение про обычку, перенесённое в карточку префа без правки. Между тем все остальные блоки префа прямо говорят обратное: governance — «выплаты направляются на привилегированные акции VGSBP», макро — «процентный доход… тянул вверх… дивидендную базу префа», бизнес-модель — «преференциалы получают дивиденды в приоритете». То есть карточка одновременно утверждает: «преф ценят за дивиденд» (3 блока) и «дивидендный метод не применим, т.к. дивидендов нет» (финблок). Это и противоречие между вкладками, и пропуск самого важного для префа метода оценки.

### Противоречие цены и fair value между блоками
- summary, абзац 1: цена префа **13,15 ₽** (MOEX ISS).
- summary, встроенная VGSB-шапка: цена **13,7 ₽** (TQBR, 01.06.2026), капитализация ~4,4 млрд.
- summary §6: «коридор консервативно ~16,5 ₽, база ~16,5 ₽… против цены 13,7 ₽ → апсайд +20%».
- JSON: base 19,3 ₽, conservative 16,6 ₽, против цены 16,89 ₽.

Четыре разных ценовых/оценочных набора в одной карточке. summary и JSON не согласованы даже по базовой fair value (16,5 vs 19,3) и по текущей цене (13,15 vs 13,7 vs 16,89).

### Грубый структурный дефект — «двойная шапка» в summary
`financials_summary.md` содержит ДВА заголовка и две вводных: сначала корректную префовую «ПАО Волгоградэнергосбыт (прив.) (VGSBP)», затем целиком вставленную обычкину «Волгоградэнергосбыт (VGSB) — Финансы и оценка» со своим ISIN, ценой обычки и текстом «Привилегированные акции VGSBP выводятся отдельно». То есть внутри карточки префа сидит нетронутый текст карточки обычки, который про этот же преф говорит в третьем лице. Для читателя это выглядит как незавершённый копипаст.

### Что сделано хорошо (качественные блоки)
- **Макро** и **Гео** для префа отредактированы осмысленно: правильный акцент, что для префа ключевое — процентный доход на кэш → дивидендная база, и что низкая ликвидность префа делает цену чувствительной к страновому риск-фону. Это настоящая префовая адаптация, а не копия.
- **Управление** честно фиксирует, что преф — основной канал дивидендов, обычка — нет. Логика «классов акций» проведена.
- **Бизнес-модель** корректно говорит «экономика едина с VGSB, различие — в правах» — это правомерно для одного эмитента.

## 3. ПО ВКЛАДКАМ

### Финансы и оценка — не выполнено для префа (2/10)
Главная и проваленная вкладка. JSON — копия обычки; коэффициент 0,96 заявлен, но не применён; дивидендный метод отключён обычкиным обоснованием; summary с двойной шапкой и расходящимися цифрами. Для префа должно было быть: цена префа как база, дивидендный метод КАК ОСНОВНОЙ (история выплат на VGSBP, уставный минимум 10% ЧП / номинал — типовая формула, дивдоходность), и fair value от дивиденда, а не P/E обычки. Ничего из этого нет.

### Бизнес-модель — корректная, но с теми же числовыми проблемами, что у обычки (6/10)
Правомерно опирается на единую экономику. Но тащит тот же конфликт выручки: таблица 36,4 млрд (2024, РСБУ) против финблока 32,0 млрд. И раздел «Отличие префов» описывает уставный приоритет общими словами («обычно в большем размере», «уставом определён минимальный размер») — без конкретной уставной формулы дивиденда префа, которая и есть суть бумаги.

### Макро — сильный, по-настоящему префовый (8/10)
Правильно вынес процентный доход на кэш как главный префовый нерв. Хорошо.

### Гео — корректный, префово-адаптированный (7/10)
Канал «низкая ликвидность префа → чувствительность к страновому фону» — точное наблюдение. Не раздут.

### Управление — честный (7/10)
Зеркалит обычку, но с правильной префовой рамкой (преф = канал дивиденда). Минус — тот же открытый вопрос по контролю (Межрегионсоюзэнерго).

## 4. ФАКТИЧЕСКИЕ ОШИБКИ / ПРОТИВОРЕЧИЯ
1. **financials.json — числа обычки** (fair value 16,6/19,3/27,5; цена 16,89; капитализация 5397) в карточке префа. Коэффициент 0,96 заявлен в summary, но не применён.
2. **Дивидендный метод = not_applicable** с обоснованием «по обычке дивидендов нет» — в карточке ПРЕФА, где дивиденд есть и является главным. Противоречит governance/макро/БМ той же карточки.
3. **Цена префа в трёх версиях**: 13,15 (summary §1), 13,7 (встроенная шапка), 16,89 (JSON-расчёты).
4. **Base fair value**: 16,5 ₽ (summary §6) vs 19,3 ₽ (JSON) — расхождение между текстом и данными.
5. **Двойной заголовок/вводная** в summary (префовая + целиком обычкина).
6. Выручка 2024: 36,4 млрд (БМ) vs 32,0 млрд (финблок) — унаследовано от обычки.
7. summary §6 называет CAPM Ke=23,6% (beta 1,0), JSON CAPM Ke=21,8% (beta 0,8) — даже параметры CAPM в тексте и данных разные.

## 5. ЧЕГО НЕ ХВАТАЕТ ДО ЭКСПЕРТНОСТИ
- **Полностью пересчитать финблок префа от цены префа**, а не множить обычку на 0,96 задним числом: котировка префа как база всех мультипликаторов, отдельная капитализация (цена префа × число префов, а не × число обычки).
- **Сделать дивидендный метод ОСНОВНЫМ**: история выплат на VGSBP, уставная формула дивиденда префа, дивдоходность к цене 13,15 ₽, fair value от капитализации дивиденда. Это и есть инвест-тезис префа — сейчас он отсутствует.
- **Свести одну цену и одну fair value** между summary и JSON; убрать встроенную обычкину шапку из summary.
- Указать **число привилегированных акций** отдельно (в shares_outstanding стоит 319,6 млн — это, похоже, обычка; для префа нужен свой объём выпуска).
- Привести конкретику уставного приоритета префа (минимальный дивиденд, условия «оживления» права голоса при невыплате) — ядро бумаги.

**Системная проблема VGSBP: финансово-оценочный слой не адаптирован под привилегированную акцию — это копия обычки (вплоть до отключённого дивидендного метода и fair value от чужой цены), отчего самый главный для префа вопрос «сколько он стоит по дивиденду» не отвечён, а карточка противоречит сама себе по цене и методу.**
