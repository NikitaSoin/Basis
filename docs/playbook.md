# Playbook — протоколы, пайплайны, технические детали

> Детали, на которые ссылается короткий `CLAUDE.md`. Читай ПО НУЖДЕ. Здесь —
> «как делать»; «что соблюдать всегда» — в CLAUDE.md; «что сейчас» — в status.md.

## Протокол крупной/незнакомой задачи (новый класс активов, большая фича)
Гендиректор проходит цикл сам, не возвращаясь к владельцу с «а как?»:

**Шаг 0 — Продуктовая постановка** (через product-analyst-fin; для второго ракурса
product-analyst-biz; создай нужного, если нет). Метод:
1. Боль/проблема — какую реальную боль частного инвестора решаем?
2. Главный вопрос (JTBD) — какую работу пользователь «нанимает» продукт сделать?
   (акция → «справедливая цена»; облигация → «доходность за риск + ставка»;
   фьючерс → «что это, плечо/экспирация, связь с портфелем»; фонд → «что внутри,
   TER, следует ли, дублирует ли». Для нового класса аналитик ОПРЕДЕЛЯЕТ свой.)
3. Варианты решения (плюсы/минусы).
4. Превращение в исполнение.
Источники: `docs/personas/` (если есть), веб, аналоги. Синтез — за тобой.

**Шаг 1 — Данные.** Что нужно и где взять. По аналогии с рабочим: список бумаг —
файлы/борды MOEX; котировки — MOEX ISS; макро/факты — веб; коэффициенты — файлы
биржи. «Нет данных» — не ответ, а начало поиска альтернатив.

**Шаг 2 — Проектирование + команда.** Продуктовое ТЗ → техническое. Доменный
аналитик данных — создавай по образцу (financial-analyst / bond-analyst):
методология → приоритет источников → 3 уровня достоверности → JSON + summary →
самопроверка.

**Шаг 3 — Реализация.** Сквозной поток до боя (данные → бэк → фронт → деплой →
проверка на живом сайте). См. Definition of Done в CLAUDE.md.

**Шаг 4 — ОТК** (обязательно перед возвратом владельцу) — см. CLAUDE.md.

Правила оркестрации: делегируй профильное; передавай субагенту только нужный
контекст; после вызова — краткий итог владельцу, не весь вывод; плохой результат
по части субагента — переформулируй/перезапусти, не «улучшай» молча; субагенты НЕ
делают git commit/push (это делаешь ты). NB: субагент, созданный в текущей сессии,
может не зарегистрироваться до перезапуска — тогда выполни его работу по его роли сам.

## Деплой (Timeweb)
- git push → Timeweb пересобирает контейнеры (фронт и бэк — отдельные приложения).
  Сборка 3-5 мин (бывает дольше). Боевой бэк: `nikitasoin-basis-e726.twc1.net`.
- Подтверждение новой версии: фронт — смена бандла `main.<hash>.js` на inbasis.ru;
  бэк — опрос боевого эндпоинта.
- Миграции и данные применяются АВТОМАТИЧЕСКИ при старте бэка (см. ниже) —
  ручная `./import_data.sh` больше не обязательна (осталась как ручной фолбэк).

## Авто-деплой данных (важно)
`app/services/asset_data.py` + `app/main.py` (lifespan):
- `ensure_migrations()` — alembic upgrade head программно при старте (не зависим
  от точки входа контейнера).
- `refresh_all_if_stale()` при старте (в фоне) и ежедневно 6:00 МСК: фьючерсы и
  фонды (дёшево, 1 запрос) — всегда; облигации (~16 мин) — если пусто/устарело (>22ч).
- Идемпотентно (upsert по SECID). CLI-скрипты `scripts/load_*.py` — для ручного/
  отладочного прогона.

## Модель данных: числа в БД, текст в файлах
- БД (`companies`, `quotes`, `index_history`, `dividends`, `company_metrics`,
  `bonds`, `futures`, `funds`, `portfolios`, `portfolio_positions`, `users`,
  `market_params`): то, что агрегируется и считается.
- Файлы `companies/<TICKER>/` (`business_model.md`, `financials.json`+summary,
  `governance/market/macro/geo.json`+summary), `bonds/<SECID>/`, `futures/<SECID>/`,
  `funds/<SECID>/` (`analysis.json`+`analysis_summary.md`): текст-интерпретация
  (едут на бой через git push).
- Числа-метрики синхронизируются в `company_metrics` (`sync_company_metrics`);
  риск-метрики пересчитываются (`recalc_risk_metrics`).

## Методики расчётов (заповедник — менять с согласия)
- Справедливая цена акции; 3 уровня достоверности (факт/оценка/суждение).
- Гибрид-бета: официальная MOEX (fortscoefficients) + расчёт Basis (поправка
  Диммсона); окна расчёта; безрисковая ставка — ОФЗ G-curve (1г).
- Облигации: риск-тир по спреду YTM к G-curve (gov/high/medium/speculative,
  пороги 250/600 б.п.); двойной рейтинг (рынок vs агентство); YTM-артефакты
  (>300%) обнуляются. Детали — docs/bonds-methodology.md.
- Фьючерсы: эффективное плечо = номинал/ГО; контанго/бэквордация по сериям.
  docs/futures-methodology.md.
- Фонды: TER в деньги; ошибка слежения (оценка Basis). docs/funds-methodology.md.

## Технические правила (полно)
- Backend-фича: модель → схема (Pydantic) → сервис → эндпоинт → тест (pytest).
- Схема БД — только Alembic-миграции (down_revision цепочкой). Массовая закачка —
  скриптом/asset_data, не миграцией.
- Секреты/токены/ключи — только `.env`, НИКОГДА не хардкодить.
- Линтеры: ruff, black. Простые решения вместо «на вырост».
- MOEX ISS — последовательно с паузами (~0.25-0.3с), батчами; на rate limit —
  пауза побольше, не долбить. Кириллица в бандле CRA экранируется в \uXXXX (норма).
- Фронт: Tailwind с префиксом `tw-`, токены в `src/styles/tokens.css`, примитивы
  на `/_design`. Динамические классы Tailwind не работают — только литеральные.
- Без рекомендаций «купить/продать» нигде.
